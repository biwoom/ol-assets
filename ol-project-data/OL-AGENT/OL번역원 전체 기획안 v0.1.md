# OL번역원 전체 기획안 v0.1

**문서명:** OL번역원 전체 기획안
**버전:** v0.1
**작성일:** 2026-06-06
**명칭 상태:** 가칭
**문서 성격:** 전체 구조 정리용 상위 기획안
**세부 문서:** 추후 별도 작성 예정

---

## 0. 문서 목적

이 문서는 OL DESK, OL Translation Lab, Paperclip 에이전트 지침 체계를 하나의 번역 프로덕션으로 통합하여 이해하기 위한 **상위 구조 문서**이다.

이 문서는 세부 개발기획서나 운영매뉴얼이 아니다.
향후 작성될 다음 문서들의 기준이 되는 **전체 구성도와 방향성**을 정리하는 역할을 한다.

```text
1. OL-DESK 개발기획문서
2. OL-Translation-Lab 데이터 구조 기획문서
3. Paperclip 에이전트 운영 기획문서
4. 번역 워크플로우 운영매뉴얼
5. 프로젝트별 번역 운영지침
```

---

## 1. 한 줄 정의

**OL번역원은 Paperclip 에이전트 생산층, OL Translation Lab 데이터층, OL DESK 인간 감수층을 결합한 로컬 중심 불교문헌 번역 프로덕션이다.**

---

## 2. 명칭에 대한 전제

`OL번역원`은 현재 가칭이다.

이 명칭은 최종 브랜드명이라기보다, 다음 세 요소를 통합적으로 지칭하기 위한 임시 상위 개념이다.

```text
OL DESK
= 인간 감수자용 로컬 대시보드

OL Translation Lab
= 번역 데이터와 산출물 저장소

Paperclip Agents
= AI 에이전트 기반 번역 생산 조직
```

향후 명칭은 프로젝트 성격에 따라 변경될 수 있다.

예비 명칭 후보:

```text
OL번역원
OL Translation Lab
OL Translation Desk
OL Scriptorium
OL Dharma Translation Lab
```

현재 단계에서는 `OL번역원`을 임시 상위명으로 사용한다.

---

## 3. 전체 구성

OL번역원은 세 개의 핵심 층위로 구성된다.

```text
OL번역원
├─ OL-DESK
│  └─ 인간 감수자용 로컬 편집 대시보드
│
├─ OL-Translation-Lab
│  └─ 번역 데이터, 원문, 차수별 번역, 검수 데이터 저장소
│
└─ Paperclip Agents
   └─ AI 에이전트 조직과 역할 지침 체계
```

각 층위의 역할은 명확히 분리한다.

```text
OL-DESK
= 보고, 검토하고, 수정하고, 확정하는 곳

OL-Translation-Lab
= 원문과 모든 산출물이 저장되는 곳

Paperclip Agents
= 에이전트들이 작업을 수행하는 조직 체계
```

## 4. 핵심 원칙

## 4.1 층위 분리 원칙

OL번역원은 다음 세 폴더를 분리하여 운영한다.

```text
ol-desk-project/
├─ ol-desk/
├─ ol-translation-lab/
└─ paperclip/
```

각 폴더의 역할은 다음과 같다.

```text
ol-desk/
= Astro 기반 OL DESK 앱 코드

ol-translation-lab/
= 실제 번역 데이터, 원문, 산출물, 검수 데이터, handoff 파일

paperclip/
= Paperclip 에이전트 초기 설정과 운영 지침문서
```

중요 원칙:

> **`paperclip/`에는 번역 산출물을 저장하지 않는다. 모든 실제 번역 산출물은 `ol-translation-lab/`에 저장한다.**

---

## 4.2 프로젝트별 독립성 원칙

각 번역 프로젝트는 서로 다른 문헌적·교학적·문체적 특성을 가진다.

따라서 용어, 각주, 태그는 `shared/`에 섞지 않는다.

```text
용어집
= 프로젝트별 관리

각주집
= 프로젝트별 관리

태그집
= 프로젝트별 관리
```

`shared/`에는 프로젝트 전체에 공통 적용할 수 있는 문서만 둔다.

```text
shared/
├─ style-guides/
├─ general-translation-principles/
├─ common-editorial-rules/
└─ templates/
```

즉, `shared/`는 누적 용어·각주·태그 저장소가 아니라 **공통 지침과 템플릿 저장소**이다.

---

## 4.3 인간 확정 우선 원칙

Paperclip 에이전트는 번역과 후보 생성을 수행할 수 있다.
그러나 최종 판단은 인간 감수자가 한다.

```text
AI 에이전트
= 생산, 후보 제안, 대조, 반영, 점검

인간 감수자
= 수정, 확정, 보류, 폐기, 최종 탈고
```

이 원칙은 기존 OL DESK 개발기획안의 핵심인 “1·2차 에이전트 번역안은 참조 자료이고, 인간의 실제 편집 공간은 3차 가번역 섹션과 4차 최종원고 섹션”이라는 구조와 연결된다. 

---

## 4.4 로컬 중심 원칙

OL번역원은 우선 로컬 중심으로 운영한다.

```text
로컬 파일 시스템
Git 관리
Astro 개발 서버
Paperclip 로컬 실행
JSON / Markdown 기반 산출
```

클라우드 서버, 다중 사용자 협업, 자동 배포, GitHub 자동 업로드는 후속 단계에서 검토한다.

## 6. OL-DESK의 위치

OL-DESK는 OL번역원의 인간 감수자 인터페이스이다.

주요 역할:

```text
1. 프로젝트 선택
2. 원본문헌 리스트 확인
3. 워크플로우 진행상태 확인
4. 1차·2차 번역문 보기
5. 3차 가번역 수정
6. 신규 용어·각주·태그 생성
7. 후보 용어·각주·태그 확정·보류·폐기
8. 3차 반영 지시 파일 생성
9. 3차 생산물 확인
10. 4차 최종원고 수정
```

OL-DESK는 `ol-translation-lab/`을 단일 데이터 소스로 읽고 쓴다.

---

## 7. OL-Translation-Lab의 위치

OL-Translation-Lab은 실제 번역 프로젝트 데이터의 저장소이다.

주요 역할:

```text
1. 원문 저장
2. 정규화 원문 저장
3. 세그먼트 데이터 저장
4. 1차·2차·3차·4차 번역 데이터 저장
5. 프로젝트별 용어 저장
6. 프로젝트별 각주 저장
7. 프로젝트별 태그 저장
8. 인간 검수 데이터 저장
9. Paperclip handoff 파일 저장
10. 최종 Markdown 저장
```

중요 원칙:

```text
프로젝트별 용어·각주·태그는 각 프로젝트 내부에 둔다.
shared/에는 공통 스타일가이드와 전체 번역지침만 둔다.
```

---

## 8. Paperclip Agents의 위치

Paperclip Agents는 AI 에이전트 기반 번역 생산 조직이다.

주요 역할:

```text
1. 원문 정리
2. 1차 번역 생성
3. 원문-번역 대조
4. 용어 후보 생성
5. 각주 후보 생성
6. 태그 후보 생성
7. 2차 번역 생성
8. 인간 감수 결과 반영
9. 3차 생산물 생성
10. 최종스캔
```

`paperclip/` 하위에는 에이전트 지침문서만 둔다.

각 에이전트 폴더에는 다음 문서가 위치한다.

```text
AGENTS.md
HEARTBEAT.md
SOUL.md
TOOLS.md
```

첨부된 Paperclip 내부 문서를 보면 CEO는 직접 실무를 수행하기보다 전략, 우선순위, 위임, 팀 구성, 승인, 장애 해소를 담당하고, 실무는 하위 에이전트에게 위임하도록 설계되어 있다. 
따라서 OL번역원의 Paperclip 구조도 CEO가 직접 번역하지 않고 A라인·B라인에 작업을 위임하는 방식으로 설계한다.

## 9. 기본 워크플로우

OL번역원의 기본 워크플로우는 네 단계로 구분한다.

---

## 9.1 1-2차 번역 워크플로우

**주도:** A00-생산관리매니저-선행
**수행:** A라인 에이전트
**목표:** OL-DESK에서 인간 감수 가능한 상태까지 문서를 준비

```text
원문 입력
→ 원문 정리
→ 세그먼트 부여
→ 1차 번역
→ 원문대조
→ 용어 후보 생성
→ 각주 후보 생성
→ 태그 후보 생성
→ 2차 번역
→ 3차 가번역 초기값 생성
```

---

## 9.2 2차 휴먼 워크플로우

**주도:** OL-DESK
**수행:** 인간 감수자
**목표:** 3차 반영을 위한 인간 수정·확정 데이터 생성

```text
1차·2차 번역 확인
→ 3차 가번역 수정
→ 신규 용어·각주·태그 생성
→ 후보 용어·각주·태그 확정
→ draft3-instructions.json 생성
```

---

## 9.3 3차 번역 워크플로우

**주도:** B00-확정관리매니저-결정
**수행:** B라인 에이전트
**목표:** 인간 감수 결과를 반영한 3차 생산물 생성

```text
draft3-instructions.json 확인
→ 3차 반영
→ 3차 생산물 생성
→ 최종스캔
→ 4차 최종원고 영역 생성 준비
```

---

## 9.4 4차 최종원고 워크플로우

**주도:** OL-DESK
**수행:** 인간 감수자
**목표:** 최종 탈고와 final.md 준비

```text
3차 생산물 확인
→ 4차 최종원고 수정
→ 최종 탈고
→ final.md export
```

---

## 10. 수정 재진입 워크플로우

4차 최종원고 또는 완료본에서 본문, 용어, 각주, 태그 수정이 발생할 수 있다.

이 경우 기존 최종본을 단순히 덮어쓰지 않고, 새 revision cycle을 연다.

```text
final_done
→ revision_needed
→ draft3_preliminary_reopened
→ draft3_requested
→ draft3_generated
→ final_review
→ final_done
```

이 구조는 첨부 구상문서의 “4차 원고로 완료된 번역본이라도 OL-DESK에서 본문·용어·각주·태그 수정 시 다시 3차 가번역본이 되고 3차번역 워크플로우가 진행된다”는 방향을 반영한 것이다.

## 11. OL-DESK 메뉴 구조 초안

OL-DESK는 대시보드형 구조를 기본으로 한다.

```text
프로젝트
문헌
번역
용어
각주
태그
에이전트
관리
```

### 프로젝트

```text
프로젝트 추가
프로젝트 전환
프로젝트별 원본 소스 디렉토리 지정
```

### 문헌

```text
원본문헌 리스트
워크플로우 진행상태
상태별 필터
개별 문헌 클릭 시 번역 페이지 이동
```

### 번역

```text
원문, 1차, 2차, 3차, 4차 번역 확인
3차 가번역 수정
4차 최종원고 수정
본문·용어·각주·태그 연동
```

### 용어

```text
개별 프로젝트 누적 용어 관리
신규 용어 추가
용어 수정
용어 보류·폐기
```

### 각주

```text
개별 프로젝트 누적 각주 관리
신규 각주 추가
각주 수정
각주 보류·폐기
```

### 태그

```text
개별 프로젝트 누적 태그 관리
신규 태그 추가
태그 수정
태그 보류·폐기
```

### 에이전트

```text
에이전트 활동 로그
산출물 검증 상태
에이전트별 작업 결과 확인
```

### 관리

```text
관리문서 생성
개선 아이디어 기록
이슈별 작업 정리
Git 변경사항 확인
```

v0.1에서는 모든 메뉴를 완성하지 않고, 우선순위를 나누어 구현한다.

## 12. Git 활용 원칙

Git은 초기 설계부터 반영한다.

다만 초기 단계에서 OL-DESK가 직접 GitHub 업로드까지 담당하지는 않는다.

```text
v0.1
- 변경 파일 인식
- 저장 이벤트 기록
- Git diff 확인을 위한 구조 준비

v0.2
- git status 표시
- 변경 파일 목록 표시
- commit message 초안 생성

v0.3 이후
- OL-DESK에서 commit 실행
- 선택적 GitHub push
- 관리문서와 이슈별 변경 이력 연동
```

## 13. 로컬 AI CLI 활용 위치

로컬 AI CLI는 Paperclip과 경쟁하는 별도 에이전트 조직이 아니다.

역할은 다음과 같다.

```text
로컬 AI CLI
= 인간 감수자 보조 도구
= 2차 번역 검토 제안
= 3차 가번역 수정 제안
= 용어 일관성 점검
= style-guide.md 기준 문체 점검
```

기본 참조 문서:

```text
project-index.md
style-guide.md
전체 번역지침
에이전트 지침문서
```

이 기능은 v0.1 핵심 기능이 아니라 후속 확장 기능으로 둔다.

# 17. 최종 정리

OL번역원은 현재 가칭이지만, 전체 구조를 잡기 위한 상위 개념으로 유용하다.

핵심 구조는 다음과 같다.

```text
Paperclip Agents
= 번역 생산과 반영을 담당하는 AI 에이전트 조직

OL Translation Lab
= 프로젝트별 번역 데이터와 산출물 저장소

OL DESK
= 인간 감수자가 수정·확정·최종 탈고를 수행하는 대시보드
```

핵심 원칙은 다음과 같다.

```text
1. paperclip/에는 지침만 둔다.
2. 실제 산출물은 ol-translation-lab/에 둔다.
3. 용어·각주·태그는 shared/가 아니라 프로젝트별로 둔다.
4. shared/에는 스타일가이드와 전체 번역지침만 둔다.
5. 인간 확정이 최종 판단 기준이다.
6. 완료본 수정은 새 revision cycle로 처리한다.
7. v0.1은 전체 자동화보다 최소 실행 가능한 번역 프로덕션을 목표로 한다.
```

이 문서의 핵심 문장은 다음이다.

> **OL번역원은 가칭이며, Paperclip 에이전트 조직, OL Translation Lab 데이터 저장소, OL DESK 인간 감수 대시보드를 결합한 로컬 중심 불교문헌 번역 프로덕션의 상위 구조이다.**

# Simplified Paperclip Translation Team Plan

## 사람 중심 번역 편집실 운영안

**문서명:** Paperclip 번역 에이전트 팀 간소화 기획안  
**전달 대상:** Paperclip Company의 번역 에이전트 팀 전체  
**기준 프로젝트:** `buddhavamsa-translation-lab/`  
**핵심 변경:** 14개 에이전트 구조를 7개 이하의 사람 중심 번역 편집실 구조로 축소한다.  
**운영 원칙:** Paperclip은 과정 관리, 파일시스템은 원고 보관, 인간 편집자는 최종 판단을 맡는다.  

---

# 1. 재기획 결론

기존 14개 에이전트 구조는 안전한 최대 구조로는 유효하지만, 실제 번역 운영의 1차 버전으로는 과하다. Paperclip이 Issue, Assignee, 댓글, 상태, 대화 기록, 감사 로그를 이미 제공하므로 파일시스템과 에이전트 조직이 같은 기록 기능을 반복할 필요가 없다.

새 운영안의 결론은 다음이다.

```text
에이전트 수:
14개 → 7개

파일 구조:
AI 단계별 산출물 중심 → 인간이 읽고 판단할 원고 중심

기록 방식:
작업 과정과 대화는 Paperclip Issue에 기록
장기 보존할 원문·번역·검수·최종본만 파일로 보관

최종 권한:
용어, 각주, 해석, 최종 문체는 인간 편집자가 확정
```

---

# 2. 새 조직도

```text
인간 편집자
└─ 00-총괄 번역 디렉터-법장
   ├─ 01-원문정리 에이전트-정안
   ├─ 02-초벌번역 에이전트-초역
   ├─ 03-대조감수 에이전트-조견
   ├─ 04-윤문번역 에이전트-윤문
   ├─ 05-검수정리 에이전트-청문
   └─ 06-최종반영 에이전트-회향
```

이 구조에서는 생산관리 매니저와 확정관리 매니저를 별도 에이전트로 두지 않는다. `00-총괄 번역 디렉터-법장`이 Paperclip Issue 상태와 대기열을 보며 작업 우선순위를 직접 조정한다. 대량 운영에서 실제 병목이 생길 때만 A/B 매니저를 다시 분리한다.

# 4. 역할 지침

## 4.1 00-총괄 번역 디렉터-법장

```text
너는 불교문헌 번역 프로젝트의 총괄 번역 디렉터다.

역할:
- Paperclip Issue를 기준으로 전체 작업 흐름을 조정한다.
- 원문정리, 초벌번역, 대조감수, 윤문번역, 검수정리, 최종반영 작업을 배정한다.
- Paperclip Issue에 남은 대화와 기록을 우선 신뢰한다.
- 파일시스템에는 장기 보존할 원고와 관리 파일만 남기도록 지시한다.
- 인간 편집자에게 필요한 판단 요청을 짧고 명확하게 정리한다.

금지:
- 직접 최종 번역을 확정하지 말 것
- 용어·각주·해석을 사람 대신 확정하지 말 것
- Paperclip Issue 기록을 파일 로그로 중복 생산하지 말 것
- final 원고를 인간 승인 없이 작성하지 말 것
```

## 4.2 01-원문정리 에이전트-정안

```text
너는 원문정리 에이전트다.

역할:
- 원본을 `원문/원본/`에 보존한다.
- 번역 작업에 필요한 정리본을 `원문/정리본/`에 만든다.
- 원문과 참고번역이 섞여 있으면 참고번역을 `참고번역/`으로 분리한다.
- 문단 ID와 기본 출처 정보를 정리한다.

금지:
- 번역하지 말 것
- 원문의 의미를 임의로 고치지 말 것
- 원문에 없는 해석을 삽입하지 말 것
```

## 4.3 02-초벌번역 에이전트-초역

```text
너는 초벌번역 에이전트다.

역할:
- `원문/정리본/`을 바탕으로 `번역/초벌/`에 초벌 번역을 작성한다.
- 직역 7, 의역 3의 비율로 원문 구조와 의미 대응을 우선한다.
- 미확정 용어는 원어 또는 한자를 병기한다.
- 신규 용어 후보는 번역문 안의 검토 메모로 남긴다.

금지:
- 문체를 위해 의미를 바꾸지 말 것
- 각주·해석 후보를 본문에 섞지 말 것
- 참고번역을 그대로 베끼지 말 것
```

## 4.4 03-대조감수 에이전트-조견

```text
너는 대조감수 에이전트다.

역할:
- 원문과 초벌 번역을 대조한다.
- 원문 누락, 의미 추가, 오역 후보, 용어 불일치를 점검한다.
- 별도 검수 파일을 남기기보다 Paperclip Issue에 감수 결과를 기록한다.
- 필요한 경우 번역문 안의 검수 메모 영역에 핵심 사항만 반영한다.

금지:
- 본문을 직접 확정 수정하지 말 것
- 문체 취향만으로 수정 지시하지 말 것
- 참고번역을 정답으로 간주하지 말 것
```

## 4.5 04-윤문번역 에이전트-윤문

```text
너는 윤문번역 에이전트다.

역할:
- 초벌 번역과 대조감수 결과를 바탕으로 `번역/윤문/`에 윤문본을 작성한다.
- 직역 3, 의역 7의 비율로 자연스러운 한국어 독서문체를 만든다.
- 원문 의미와 문단 대응을 유지한다.
- 미확정 용어·각주·해석은 본문에 확정 반영하지 않는다.

금지:
- 원문에 없는 정서, 교훈, 해석을 추가하지 말 것
- 미확정 용어를 확정된 것처럼 쓰지 말 것
- 최종본을 작성하지 말 것
```

## 4.6 05-검수정리 에이전트-청문

```text
너는 인간 검수를 준비하는 에이전트다.

역할:
- 윤문본에서 인간 편집자가 판단해야 할 용어·각주·해석 후보를 정리한다.
- `검수/검수대기/`에 사람이 바로 볼 수 있는 검수본을 만든다.
- 후보는 채택, 보류, 삭제를 선택할 수 있는 형태로 제시한다.
- Paperclip Issue에 검수 요청 요약을 남긴다.

금지:
- 후보를 확정하지 말 것
- 용어집과 편집결정을 직접 최종 갱신하지 말 것
- 사람의 판단이 없는 항목을 06-최종반영 에이전트-회향에게 넘기지 말 것
```

## 4.7 06-최종반영 에이전트-회향

```text
너는 인간 편집자의 확정 판단을 반영하는 에이전트다.

역할:
- 인간 편집자가 확정한 용어·각주·해석만 반영한다.
- 반영 완료본을 `검수/반영완료/`에 둔다.
- 최종 원문대조와 반영 누락 여부를 점검한다.
- 인간 편집자가 최종 탈고할 수 있도록 `번역/최종/` 후보를 준비한다.

금지:
- 새 용어·각주·해석을 임의로 추가하지 말 것
- 의미를 재해석하지 말 것
- 인간 승인 없이 최종본을 확정하지 말 것
```

# 5. 사람 중심 폴더 구조

새 파일 구조는 다음을 기본으로 한다.

```text
buddhavamsa-translation-lab/
├─ 관리/
│  ├─ 전체진행표.md
│  ├─ 번역원칙.md
│  ├─ 편집결정.md
│  └─ 용어집.md
│
├─ 원문/
│  ├─ 원본/
│  └─ 정리본/
│
├─ 참고번역/
├─ 번역/
│  ├─ 초벌/
│  ├─ 윤문/
│  └─ 최종/
│
├─ 검수/
│  ├─ 검수대기/
│  └─ 반영완료/
│
├─ 보류/
└─ 운영기록/
```

`운영기록/`은 Paperclip 도입 이전 또는 구조 전환 이전의 계획, 보고, 큐 파일을 보관하는 장소다. 새 작업에서는 원칙적으로 Paperclip Issue를 운영 기록으로 사용한다.

```text
batch-plans/
```

# 7. 문서별 최소 산출물

문서 하나에 필요한 파일은 원칙적으로 다음으로 제한한다.

```text
원문/원본/{문서ID}.md
원문/정리본/{문서ID}.md
번역/초벌/{문서ID}.md
번역/윤문/{문서ID}.md
검수/검수대기/{문서ID}.md
검수/반영완료/{문서ID}.md
번역/최종/{문서ID}.md
```

참고번역이 있는 문서만 다음을 추가한다.

```text
참고번역/{문서ID}.md
```

검수 메모는 별도 로그 파일로 분산하지 않는다. 사람이 확인해야 하는 핵심 검수 사항은 `검수/검수대기/{문서ID}.md`에 모으고, 과정 대화는 Paperclip Issue에 남긴다.

# 8. 상태값

Paperclip Issue의 상태 또는 태그는 다음 정도로 단순화한다.

```text
원문대기
원문정리완료
초벌완료
대조감수완료
윤문완료
인간검수대기
인간검수완료
반영완료
최종대기
최종완료
보류
오류
```

파일시스템 안에서 별도 큐 파일을 운영하지 않는다. 다만 인간 편집자가 전체 흐름을 빠르게 보아야 하므로 `관리/전체진행표.md`는 유지한다.

---

# 9. 최종 원칙

```text
Paperclip = 과정 관리
파일시스템 = 원고 보관
인간 편집자 = 확정 판단
AI 에이전트 = 초안 생산과 검수 보조
```

이 기획안의 목적은 AI가 보기 좋은 구조를 만드는 것이 아니라, 인간 편집자가 원문, 번역, 검수, 최종본을 직관적으로 확인할 수 있는 번역 편집실을 만드는 것이다.

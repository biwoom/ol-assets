## 전략 A 핵심 원칙

shadcn/ui의 디자인 본질은 **"눈에 띄지 않는 디자인"** 입니다. 사용자가 디자인 자체에 집중하지 않고 **컨텐츠에 몰입**할 수 있게 만드는 것이 목표입니다.

```
모던   → 무채색 베이스 + 정확한 1px 보더
심플   → 장식 제거, 충분한 여백, 명확한 위계
세련   → 미세한 그림자, 정밀한 간격, 일관된 radius
질리지 않음 → 개인 취향이 강한 컬러 사용 안 함
```

## 디자인 시스템 정의

### 1. 색상 토큰 (HSL 기반)

shadcn의 공식 "neutral" 팔레트를 그대로 채택합니다. 라이트/다크 모드 양쪽 정의.

```css
:root {
  --background:         0 0% 100%;
  --foreground:         0 0% 3.9%;
  --card:               0 0% 100%;
  --card-foreground:    0 0% 3.9%;
  --popover:            0 0% 100%;
  --popover-foreground: 0 0% 3.9%;
  --primary:            0 0% 9%;
  --primary-foreground: 0 0% 98%;
  --secondary:          0 0% 96.1%;
  --secondary-foreground: 0 0% 9%;
  --muted:              0 0% 96.1%;
  --muted-foreground:   0 0% 45.1%;
  --accent:             0 0% 96.1%;
  --accent-foreground:  0 0% 9%;
  --destructive:        0 84.2% 60.2%;
  --destructive-foreground: 0 0% 98%;
  --border:             0 0% 89.8%;
  --input:              0 0% 89.8%;
  --ring:               0 0% 3.9%;
  --radius:             0.5rem;
}

.dark {
  --background:         0 0% 3.9%;
  --foreground:         0 0% 98%;
  /* ... 다크 모드 토큰 */
}
```

### 2. 타이포그래피

```
시스템 폰트 스택 (shadcn 기본):
  -apple-system, BlinkMacSystemFont, "Segoe UI",
  Roboto, Helvetica, Arial, sans-serif

한국어 보강:
  + "Pretendard Variable", "Apple SD Gothic Neo"

코드/모노:
  ui-monospace, "SF Mono", "Cascadia Code",
  "Roboto Mono", monospace
```

장식적 폰트(Playfair Display, Fraunces) 모두 제거. 시스템 폰트로 통일.

### 3. 간격 시스템

shadcn은 Tailwind 4-base 시스템을 따릅니다. 픽셀이 아닌 토큰으로:

```
--space-1: 0.25rem (4px)
--space-2: 0.5rem  (8px)
--space-3: 0.75rem (12px)
--space-4: 1rem    (16px)
--space-6: 1.5rem  (24px)
--space-8: 2rem    (32px)
--space-12: 3rem   (48px)
```

### 4. radius 시스템

```
--radius-sm: calc(var(--radius) - 4px)  /* 4px */
--radius-md: calc(var(--radius) - 2px)  /* 6px */
--radius:    0.5rem                      /* 8px */
--radius-lg: calc(var(--radius) + 2px)  /* 10px */
```

## 컴포넌트 매핑 표

OL의 모든 UI 요소를 shadcn 컴포넌트에 매핑합니다.

|OL 현재|shadcn 컴포넌트|비고|
|---|---|---|
|헤더 네비 버튼|`NavigationMenu` 패턴|호버 시 underline|
|검색 입력|`Input` + `Command` 팔레트|Cmd+K|
|사이드바|자체 + `ScrollArea`|shadcn에 정확한 매치 없음|
|그룹 항목|`Button variant="ghost"`|활성 시 secondary|
|칸반 컬럼|`Card`|rounded-lg + border|
|칸반 카드|`Card` (작은 변형)|hover 시 미세한 ring|
|카드 그리드|`Card` 그리드|grid + gap|
|리스트 테이블|`Table`|얇은 row 보더|
|카드 편집 모달|`Dialog`|중앙 fixed 모달|
|온보딩 모달|`Dialog`|동일 패턴|
|검색 결과|`Command` 팔레트|키보드 네비|
|태그 다중 필터|`Popover` + `Checkbox`|정통 패턴|
|컬러 피커|`Popover`|그리드|
|셀렉트 박스|`Select`|커스텀 트리거|
|우선순위 버튼|`ToggleGroup`|3개 토글|
|학습 상태 버튼|`ToggleGroup`|동일|
|토스트|`Sonner` 스타일|우하단|
|변경이력 뱃지|`Badge` variants|의미별 색상|
|프로필 아바타|`Avatar`|원형 + fallback|
|파일 서명 카드|`Card` + 그라데이션 제거|미니멀|
|타임라인|자체 + `Separator`|shadcn 미제공|

## 작업 로드맵

### Phase 1 — 기반 시스템 구축 (1차 작업)

새 디자인 토큰과 핵심 컴포넌트만 먼저 적용합니다.

**작업 내용**

- 기존 한지/먹/단청 변수 전부 제거
- shadcn HSL 토큰 시스템 도입
- 라이트/다크 모드 변수 정의
- 폰트 시스템 단순화
- `.btn`, `.input`, `.card` 등 기본 클래스 재작성
- 헤더, 카드 모달만 우선 적용해서 효과 확인

**검증 포인트**

- 기존 기능 정상 작동
- 헤더와 모달이 shadcn 느낌으로 변환됨
- 색상이 무채색으로 통일됨

---

### Phase 2 — 레이아웃 골격 (2차 작업)

**작업 내용**

- 헤더 재구성: 깔끔한 보더, 미니멀 네비
- 사이드바 재구성: shadcn ScrollArea 패턴
- 메인 컨텐츠 영역 정리
- 검색바를 `Command` 팔레트 스타일로 (Cmd+K 트리거)
- 다크모드 토글 버튼 추가

**검증 포인트**

- 좌우 비율과 여백이 균형
- 사이드바 항목이 ghost 버튼처럼 동작
- 다크 모드 전환이 자연스러움

---

### Phase 3 — 데이터 뷰 재디자인 (3차 작업)

**작업 내용**

- 칸반 컬럼을 `Card` 컴포넌트로
- 칸반 카드를 더 작은 `Card` 변형으로
- 카드 그리드 뷰를 정통 `Card` 그리드로
- 리스트 뷰를 shadcn `Table` 스타일로
- 호버/활성/포커스 상태 정밀화

**검증 포인트**

- 드래그앤드롭 정상 동작
- 정렬/필터 정상 동작
- 모든 호버 상태 일관성

---

### Phase 4 — 인터랙션 컴포넌트 (4차 작업)

**작업 내용**

- 카드 편집 모달을 `Dialog` 스타일로
- 온보딩 모달 동일 적용
- 우선순위/학습상태 버튼을 `ToggleGroup` 스타일로
- 태그 필터를 `Popover` + `Checkbox` 패턴으로
- 컬러 피커를 깔끔한 `Popover` 그리드로
- 검색 결과를 `Command` 팔레트로

**검증 포인트**

- 키보드 네비게이션 (Tab, Enter, Esc, ↑↓)
- focus ring이 모든 인터랙티브 요소에 표시
- Cmd+K로 검색 즉시 활성화

---

### Phase 5 — 변경이력 + 프로필 (5차 작업)

**작업 내용**

- 파일 서명 카드를 절제된 shadcn 스타일로 (印 도장 모티프 제거)
- 타임라인을 더 깔끔한 라인 + 점 패턴으로
- 변경이력 액션 뱃지를 `Badge` variants로 통일
- 프로필 페이지를 카드 기반 폼으로
- Avatar 컴포넌트 정통 스타일

**검증 포인트**

- 정보 위계가 명확
- 색상 사용 절제됨
- 한자 도장 등 동양적 장식 모두 제거됨

---

### Phase 6 — 마이크로 인터랙션 (6차 작업)

**작업 내용**

- 모든 transition 통일 (`transition-colors duration-150`)
- focus-visible ring 표준화
- 토스트를 Sonner 스타일로 우하단 이동
- 빈 상태 일러스트레이션 추가
- 로딩/스켈레톤 상태 (있다면)

**검증 포인트**

- 모든 상호작용이 매끄럽게 느껴짐
- 시각적 피드백이 명확하지만 과하지 않음

## 기타

**A) 다크 모드 포함 **

shadcn의 핵심 가치 중 하나입니다. 헤더 우측에 토글 버튼을 넣고 사용자가 라이트/다크를 직접 선택할 수 있게 할까요? (기본값은 시스템 설정 따라가기)

**B) 아이콘 사용 정책**

shadcn 미니멀리즘에 맞추려면 이모지 대신 lucide-react 같은 아이콘 라이브러리 스타일 SVG 아이콘이 더 어울립니다. 단일 HTML이라 외부 라이브러리는 못 쓰니, **인라인 SVG 아이콘**을 직접 정의해서 사용

## 4. 빌드 산출물 검증

### 4.1 확인된 사실

- `[dist/ol-atlas_v0.0.4.html](/Users/damjin/Projects/ol-project/ol-atlas-v0.0.4/dist/ol-atlas_v0.0.4.html)` 안에 `window.__OL_BUILD__={"version":"0.0.4","schemaVersion":9,...}`가 들어 있다.
- 동일 파일 안에 `__LOADED_DATA_B64__ = '__INIT_DATA_B64__'` 플레이스홀더가 들어 있다.
- 동일 파일은 CSS를 `<style>`로 인라인하고, JS 번들을 `<script>`로 인라인한 단일 HTML이다.
- `AUTHOR_BUNDLE` 마커는 보이지 않는다.
- 읽기 모드용 버튼과 스타일 규칙이 산출물에 반영돼 있다.
- 편집자 시스템과 About 히스토리 관련 코드가 산출물에 반영돼 있다.

### 4.2 산출물 해석

- 이 빌드는 정적 배포용 “자기완결형 HTML”이다.
- 앱 실행 시 별도 번들 파일을 요구하지 않는다.
- 저장/불러오기 데이터는 `__LOADED_DATA_B64__`를 통해 파일 내부에 내장된다.
- dev 서버용 자산과 prod 산출물은 같은 소스에서 갈라지지만, 목적이 다르다.

## 5. 현재 구현의 실제 구조

### 5.1 상태와 스키마

- 현재 스키마 버전은 `9`다.
- `meta`에는 `editors`, `saveLog`, `actLog`, `currentEditorId`가 있다.
- `cards`에는 `acts`가 있다.
- `makeDefault()`의 기본 버전 문자열은 아직 `0.0.1`이다.

### 5.2 런타임 흐름

- 진입점은 `[src/main.js](/Users/damjin/Projects/ol-project/ol-atlas-v0.0.4/src/main.js)`다.
- `main.js`는 CSS, 뷰 컴포넌트, 검색, 이벤트를 순서대로 로드한다.
- `[src/data/init.js](/Users/damjin/Projects/ol-project/ol-atlas-v0.0.4/src/data/init.js)`가 저장 상태를 읽고, 마이그레이션하고, 정규화하고, 필요하면 내장 데이터를 우선 적용한다.
- 라우팅은 `[src/core/router.js](/Users/damjin/Projects/ol-project/ol-atlas-v0.0.4/src/core/router.js)`가 담당한다.
- 모든 상태 변경은 `[src/core/action.js](/Users/damjin/Projects/ol-project/ol-atlas-v0.0.4/src/core/action.js)`를 통해 들어간다.

### 5.3 문서뷰

- 읽기 모드 토글은 문서뷰 전용이다.
- 헤더와 사이드바는 읽기 모드에서 숨기되 hover 또는 edge-touch로 다시 드러나게 설계돼 있다.
- 모바일 대응도 포함돼 있다.
- 문서뷰 편집과 읽기 모드는 같은 파일 안에 공존한다.

### 5.4 편집 기록

- 카드 단위의 생성/수정/삭제/복구 기록은 `card.acts`에 남는다.
- 영구삭제 시에는 `meta.actLog`로 아카이브되어 기록이 사라지지 않는다.
- About 탭의 “편집 기록”은 이 데이터를 편집자 단위로 재조합해 보여준다.

## 6. 문서 대비 차이점

- 문서에서 예시로 든 `schemaVersion 7`은 현재 `9`로 올라갔다.
- 문서에서 제안한 `core/history.js` 중심 구조는 실제로는 구현되지 않았고, 현재는 About 탭 UI가 히스토리 기능을 담당한다.
- 문서의 dev 서버 예시는 `dist/ol-atlas_dev.html` 고정 파일명을 가정했지만, 실제 구현은 `dist/index.html`과 `bundle.js`를 esbuild 서버가 서빙하는 방식이다.
- 새 파일의 기본 메타 버전은 여전히 `0.0.1`이다. 패키지 버전 `0.0.4`와는 분리돼 있다.
- `package.json` 버전, 빌드 메타, 산출물 파일명이 서로 일치하도록 정리돼 있다.

## 7. 현재 상태 총평

- 이 저장소는 단순한 초안이 아니라, 문서 `v0.0.1`부터 `v0.0.4`까지의 요구를 순차적으로 흡수해 온 누적형 코드베이스다.
- 프로덕션 산출물 `[dist/ol-atlas_v0.0.4.html](/Users/damjin/Projects/ol-project/ol-atlas-v0.0.4/dist/ol-atlas_v0.0.4.html)`은 현재 소스의 핵심 기능을 모두 포함하는 단일 HTML이다.
- 기능 축은 문서뷰, 편집자/히스토리, 카드 행위 기록, dev 서버, 저장/가져오기, 사이드바 필터링으로 정리된다.
- 남은 차이점은 주로 문서와 구현 방식의 세부 불일치, 오래된 주석, 기본 버전 문자열의 하위 버전 유지 정도다.

## 1. 구현 가능성 판단

### 질문: About 폼 → 홈 랜딩 데이터 흐름이 가능한가?

**결론: 완전히 구현 가능하며, 올바른 방향입니다.**

현재 v0.0.4의 `meta` 객체는 `title`, `version`, `fileId`, `created`, `editors`, `saveLog`, `actLog`, `currentEditorId`를 갖습니다. 이미 `meta.title`과 `meta.version`은 About 폼에서 편집 → `dispatch(META_UPDATE({title, version}))` → `state.meta`에 저장 → 사이드바 `.sb-meta-line`에서 읽어 출력하는 흐름으로 작동 중입니다.

`bookInfo` 필드를 `meta` 하위에 추가하면 완전히 동일한 패턴으로 작동합니다:

```
About 폼(입력) → dispatch(META_UPDATE) → state.meta.bookInfo 저장
→ 홈 랜딩(출력): S.meta.bookInfo 읽어 렌더링
→ 사이드바 sb-meta: 책제목 + 버전 표시 (기존 유지)
```

"데이터 흐름 역방향"이라고 지적했던 이유는 About이 발행물을 "제어"하는 관계가 된다는 의미였는데, 실제로는 **About이 메타데이터 편집 인터페이스이고 홈은 그 데이터를 표시하는 뷰**이므로 이는 완전히 자연스러운 MVC 패턴입니다. 이전 판단을 철회합니다.

## 3. bookInfo 스키마 설계

`state.meta.bookInfo` 에 아래 필드를 추가합니다. 모두 선택적(optional)이며 미입력 시 기본값 또는 빈 문자열 처리합니다.

```js
meta.bookInfo = {
  bookTitle:    "",   // 책 제목 (예: "붓다스토리")
  subtitle:     "",   // 부제목
  author:       "",   // 저자
  translator:   "",   // 역자 (없으면 빈 문자열)
  publisher:    "",   // 출판/제작처
  publishedAt:  "",   // 초판 출판일 (ISO date string or 자유형식)
  revisedAt:    "",   // 최종 개정일
  bookVersion:  "",   // 책 버전 (예: "1.2.3") ← meta.version 과 별도
  description:  "",   // 한 줄 소개 (랜딩 hero sub에 표시)
  coverColor:   "",   // 표지 배경색 (hex or hsl, 선택)
  language:     "ko", // 언어 코드
  isbn:         "",   // ISBN (있다면)
}
```

**`meta.title`과 `meta.bookInfo.bookTitle`의 관계:**  
`meta.title`은 기존 파일명 생성·사이드바 표시에 사용되는 "파일 제목"으로 유지합니다. `bookInfo.bookTitle`은 랜딩 페이지에 표시되는 "책 제목"으로 분리합니다. 사용자가 두 필드를 동기화할 필요 없도록, About 폼에서 `bookTitle`을 저장할 때 `meta.title`도 함께 갱신하는 옵션을 제공합니다.

## 4. 홈 랜딩 페이지 구조 설계

### 4.1 헤더 (Mainline Astro 스타일)

```
[OL 로고] ─────────── [편집] [문서] [About] [테마토글]
```

- **기본 상태**: 홈 랜딩에서만 이 헤더가 보임. 편집 뷰로 이동하면 기존 헤더로 교체.
- **형태**: 상단 중앙 floating pill형 네비바. 배경 흰색(라이트)/어두운색(다크), 1px border, border-radius: 9999px, box-shadow 미세하게.
- **위치**: `position: sticky; top: 1rem;` 또는 `fixed + margin`
- **버튼**: ghost 스타일, 활성 뷰는 underline indicator
- **테마 토글**: 라이트 → 다크 → 독서(세피아) → 라이트 순환

### 4.2 랜딩 섹션 구성

### Task 4 — META_UPDATE 액션: bookInfo 지원

**파일**: `src/actions/settings-actions.js` (또는 META_UPDATE 정의 위치)

현재 `META_UPDATE` 액션이 `meta` 최상위 필드만 업데이트한다면, `bookInfo` 딥 머지를 지원하도록 확장:

```js
case 'META_UPDATE': {
  const patch = action.payload;
  const newMeta = { ...state.meta, ...patch };
  // bookInfo는 딥 머지
  if (patch.bookInfo) {
    newMeta.bookInfo = { ...state.meta.bookInfo, ...patch.bookInfo };
  }
  return { ...state, meta: newMeta };
}
```

## 주의사항

1. **기존 meta.title 보존**: `bookInfo.bookTitle`은 별도 필드. `meta.title`을 건드리지 않는 것이 기본값 (동기화는 opt-in).
2. **bookInfo textarea**: `description` 필드는 `<textarea>` 사용. 기존 `<input>` escape 함수가 아닌 `textContent` 방식으로 값 설정할 것.
3. **CSS grid conflict**: `.about-bookinfo-grid`는 `.about-origin-body`와 다른 클래스. 충돌 없음.
4. **토글 아이콘 순서**: Phase 2에서 테마 토글을 3단으로 바꿀 때 기존 `theme-toggle` 이벤트 리스너 교체 필요. Phase 1에서는 기존 2단 토글 유지.

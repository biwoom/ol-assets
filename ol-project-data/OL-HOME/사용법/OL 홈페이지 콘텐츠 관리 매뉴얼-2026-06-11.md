# OL 홈페이지 콘텐츠 관리 매뉴얼 v1

> **대상**: OL 프로젝트 콘텐츠 작성자와 운영자  
> **기준일**: 2026-06-11  
> **기술 스택**: Astro 6 + GitHub Pages

---

## 0. 핵심 개념

OL HOME은 Astro 기반 정적 사이트입니다. 콘텐츠는 `src/content/` 아래 Markdown 파일로 관리하고, `npm run build`를 실행하면 정적 HTML이 `dist/`에 생성됩니다.

각 콘텐츠 파일은 상단의 frontmatter에 메타데이터를 적고, 그 아래에 본문을 작성합니다.

```markdown
---
title: "제목"
published: true
---

본문을 작성합니다.
```

### 현재 운영 구조

현재 OL HOME의 주요 콘텐츠 컬렉션은 다음과 같습니다.

```
src/content/
├── blog/       ← 작업 기록, 개발 기록, 제작 노트
├── text/       ← 경전·논서·선어록·강의록을 번역하고 주석한 문헌 콘텐츠
├── story/      ← 불교의 인물·설화·가르침을 이야기 형식으로 재구성한 서사 콘텐츠
├── design/     ← 인포그래픽, 도식, 이미지 자료
├── pages/      ← 브랜드 문서 등 상설 문서
├── entities/   ← 인물, 장소, 개념 등 엔티티
└── ontology/   ← 온톨로지 정의
```

`book/`과 `works/` 라우트는 호환을 위해 남아 있지만, 현재 신규 콘텐츠 운영의 중심은 `text/`, `story/`, `design/`입니다.

### 자산 관리 원칙

사람이 직접 관리하는 원본은 `src/content` 안에 둡니다.

```
src/content/design/{slug}/assets/
src/content/story/{series}/assets/
```

빌드 전에 `scripts/sync-content-assets.mjs`가 `assets/` 폴더를 찾아 `public/generated/`로 복사합니다.

```
src/content/design/two-perspectives/assets/index.html
→ public/generated/design/two-perspectives/index.html

src/content/story/buddha-story/assets/index.html
→ public/generated/story/buddha-story/index.html
```

`public/generated/`는 빌드 산출물입니다. 직접 원본처럼 관리하지 않습니다.

현재 자동 동기화 대상은 다음 두 컬렉션입니다.

- `design`
- `story`

TEXT의 이미지는 현재 Astro content asset 방식으로 문서 또는 시리즈 폴더 안에서 상대 경로를 사용합니다.

### 빌드와 미리보기

```bash
npm run dev       # 로컬 개발 서버
npm run build     # 자산 동기화 + Astro 빌드 + Pagefind 인덱싱
npm run preview   # 빌드 결과 미리보기
```

`npm run build`는 내부적으로 다음을 실행합니다.

```bash
npm run sync:assets
astro build
npx pagefind --site dist
```

### 태그 원칙

현재 TEXT와 STORY의 읽기 UI는 `tags`를 prefix 방식으로 해석합니다.

```yaml
tags:
  - "인물/아나타삔디까"
  - "장소/라자가하-왕사성"
  - "개념/연기"
tagAliases:
  "인물/아나타삔디까": ["Anāthapiṇḍika", "급고독장자"]
```

- `/` 앞부분은 prefix입니다. 예: `인물`, `장소`, `개념`
- `/` 뒷부분은 실제 태그명입니다.
- 시리즈 페이지는 현재 문서에 존재하는 prefix만 자동으로 분류해 필터 버튼을 만듭니다.
- 여러 태그를 선택하면 AND 조건으로 문서가 좁혀집니다.

DESIGN과 BLOG에는 기존 `prefixTags`도 계속 사용합니다.

```yaml
prefixTags:
  - "kind:infographic"
  - "topic:연기"
  - "format:html"
```

`prefixTags`는 관리용·확장용 태그입니다. TEXT/STORY의 사용자 필터는 현재 `tags`의 `prefix/name` 구조를 주로 사용합니다.

### 권리·라이선스 원칙

직접 만든 OL 저작물은 특별한 사유가 없으면 다음을 기본값으로 사용합니다.

```yaml
license: "CC0"
rightsHolder: "비움"
```

외부 저작물, 강의록, 논문, 번역물은 권리 정보를 명확히 적습니다.

```yaml
authors: ["도남(圖南) 김성철"]
rightsHolder: "도남(圖南) 김성철"
license: "CC BY 4.0"
licenseUrl: "https://creativecommons.org/licenses/by/4.0/"
copyrightNotice: "© 도남(圖南) 김성철. 인용과 재배포는 라이선스 조건과 출처 표기를 따릅니다."
```

라이선스가 불명확한 자료에는 임의로 CC 라이선스를 붙이지 않습니다.

## 1. TEXT 콘텐츠

TEXT는 경전·논서·선어록·강의록을 번역하고 주석한 문헌 콘텐츠입니다.

TEXT는 메인 페이지와 시리즈 페이지가 분리되어 있습니다.

- `/text/`: 시리즈 카드 목록
- `/text/{series}/`: 시리즈 개요, 태그 필터, 문서 검색 패널
- `/text/{series}/{slug}/`: 실제 문서 상세 페이지

### 1.1 폴더 구조

TEXT는 시리즈 폴더를 원칙으로 합니다.

```
src/content/text/
└── body-mind-transformation/
    ├── index.md              ← 시리즈 메타데이터
    ├── assets/               ← 시리즈 대표 이미지 등
    │   ├── thumb.jpg
    │   └── preview1.jpg
    └── part/
        └── 01-six-steps.md   ← 실제 문서
```

시리즈 카드에 필요한 정보는 `index.md`에 작성합니다.

### 1.2 TEXT 시리즈 index.md 예시

```yaml
---
title: "몸과 마음의 전환"
kind: series
series: "몸과 마음의 전환"
seriesSlug: "body-mind-transformation"
seriesOrder: 10
description: "불교명상 기초수행을 문헌 형식으로 정리한 시리즈."
thumbnailAsset: "assets/thumb.jpg"
coverAsset: "assets/preview1.jpg"
status: published
tags:
  - "개념/불교명상"
  - "개념/정화"
prefixTags:
  - "kind:text"
  - "topic:불교명상"
license: "CC0"
published: true
---
```

중요한 점:

- `kind: series`를 지정합니다.
- 이 파일은 문서 본문이 아니라 시리즈 메타데이터입니다.
- TEXT 메인 페이지의 카드 정보는 이 파일에서 가져옵니다.

### 1.3 TEXT 문서 예시

```yaml
---
title: "몸과 마음의 정화 여섯 단계"
kind: document
series: "몸과 마음의 전환"
seriesSlug: "body-mind-transformation"
seriesOrder: 10
part: "기초수행"
partSlug: "basic-practice"
partOrder: 10
group: "정화"
groupOrder: 10
chapter: 1
order: 1
status: published
tags:
  - "개념/불교명상"
  - "개념/정화"
  - "형식/수행안내"
tagAliases:
  "개념/정화": ["purification", "몸과 마음의 정화"]
authors: ["비움"]
license: "CC0"
published: true
---

본문을 작성합니다.
```

### 1.4 TEXT 필드 설명

| 필드 | 설명 |
|------|------|
| `title` | 제목 |
| `kind` | `series` 또는 `document` |
| `series` | 시리즈 표시명 |
| `seriesSlug` | URL에 쓰는 시리즈 slug |
| `seriesOrder` | 시리즈 정렬 순서 |
| `part` | 시리즈 안의 대분류 |
| `partSlug` | part slug |
| `partOrder` | part 정렬 순서 |
| `group` | part 안의 소분류 |
| `groupOrder` | group 정렬 순서 |
| `chapter` | 독자에게 보이는 장 번호 |
| `order` | 같은 group 안의 실제 정렬 순서 |
| `status` | `draft`, `revising`, `ready`, `published` |
| `tags` | 표시·필터용 태그. TEXT에서는 `prefix/name` 방식 권장 |
| `tagAliases` | 태그의 다른 이름, 영문명, 한역명 |
| `prefixTags` | 관리용 prefix 태그 |
| `authors` | 저자 |
| `license` | 라이선스 |
| `published` | 공개 여부. 기본값은 `false` |

### 1.5 TEXT 탐색 기능

TEXT 시리즈 페이지는 다음 기능을 제공합니다.

- prefix별 태그 필터
- 다중 태그 AND 필터
- 태그 결과 문서 목록
- 시리즈 내부 문서 목록

TEXT 문서 상세 페이지는 다음 기능을 제공합니다.

- breadcrumb
- On this page 드롭다운 목차
- Copy 드롭다운
  - 링크 복사
  - 본문 마크다운 복사
- 이전 문서 / 다음 문서

## 2. STORY 콘텐츠

STORY는 불교의 인물·설화·가르침을 이야기 형식으로 재구성한 서사 콘텐츠입니다.

STORY는 책 한 권처럼 읽는 시리즈 리더를 사용합니다.

- `/story/`: 시리즈 카드 목록
- `/story/{series}/`: 시리즈 상세 리더

STORY의 개별 문서는 별도 URL로 이동하지 않고, 시리즈 페이지 안에서 PART 탭과 문서 선택으로 전환됩니다.

### 2.1 폴더 구조

```
src/content/story/
└── buddha-story/
    ├── index.md              ← 시리즈 서지정보
    ├── assets/
    │   └── index.html        ← 완성 HTML 등 시리즈 자산
    ├── part-1/
    │   ├── 02-palace-and-four-sights.md
    │   └── 03-renunciation.md
    └── part-2/
        ├── 01-awakening.md
        ├── 02-first-sermon.md
        └── 03-community.md
```

`index.md`도 STORY 컬렉션의 entry입니다. 현재 시리즈의 서지정보 탭과 대표 정보에 사용됩니다.

### 2.2 STORY 시리즈 index.md 예시

```yaml
---
title: "붓다 스토리"
subtitle: "부처님의 일대기"
series: "붓다 스토리"
seriesSlug: "buddha-story"
seriesOrder: 10
part: "서지정보"
partSlug: "bibliography"
partOrder: 0
group: "서지정보"
groupSlug: "bibliography"
groupOrder: 0
category: "인물 · 행적"
version: "v1.0"
status: draft
publishedAt: 2026-06-11
htmlAsset: "index.html"
description: "싯다르타 고타마의 생애를 이야기 형식으로 재구성한 시리즈."
tags:
  - "인물/붓다"
  - "개념/불전"
prefixTags:
  - "kind:story"
  - "topic:붓다전기"
license: "CC0"
rightsHolder: "비움"
lang: "ko"
published: true
---
```

### 2.3 STORY 문서 예시

```yaml
---
title: "출가"
series: "붓다 스토리"
seriesSlug: "buddha-story"
seriesOrder: 10
part: "탄생과 출가"
partSlug: "part-1"
partOrder: 10
group: "출가"
groupSlug: "renunciation"
groupOrder: 20
chapter: 3
order: 3
category: "붓다전기"
version: "v1.0"
status: draft
publishedAt: 2026-06-11
htmlAsset: "index.html"
description: "싯다르타가 궁을 떠나는 장면."
tags:
  - "인물/싯다르타"
  - "장소/카필라밧투"
  - "개념/출가"
tagAliases:
  "장소/카필라밧투": ["Kapilavatthu", "카필라성"]
license: "CC0"
lang: "ko"
published: true
---

본문을 작성합니다.
```

### 2.4 STORY 필드 설명

| 필드 | 설명 |
|------|------|
| `title` | 문서 제목 |
| `subtitle` | 시리즈 또는 문서 부제 |
| `series` | 시리즈 표시명 |
| `seriesSlug` | URL에 쓰는 시리즈 slug |
| `seriesOrder` | 시리즈 정렬 순서 |
| `part` | PART 탭 이름 |
| `partSlug` | PART 탭 식별자 |
| `partOrder` | PART 정렬 순서 |
| `group` | 사이드바 내부 그룹 |
| `groupSlug` | group 식별자 |
| `groupOrder` | group 정렬 순서 |
| `chapter` | 독자에게 보이는 장 번호 |
| `order` | 같은 group 안의 실제 정렬 순서 |
| `htmlAsset` | `assets/`에서 복사되는 HTML 파일명. 현재 스키마상 필수 |
| `publishedAt` | 발행일. 현재 스키마상 필수 |
| `tags` | 표시·필터용 태그. STORY에서는 `prefix/name` 방식 권장 |
| `tagAliases` | 태그 별칭 |
| `published` | 공개 여부. 기본값은 `true` |

### 2.5 STORY 리더 기능

데스크탑:

- 상단 시리즈 제목
- 서지정보 탭
  - 커버
  - 문서목록
  - 서지정보
- PART 탭
  - 왼쪽 사이드바: 해당 PART의 group > document 목록
  - 오른쪽 본문: 한 번에 하나의 문서만 표시
- prefix별 태그 필터와 검색 패널
- On this page 목차
- Copy 드롭다운
- 이전 문서 / 다음 문서

모바일:

- `[목록] [태그] [On this page] [Copy]` 도구 버튼
- 목록 버튼에서 현재 PART 문서 전환
- 태그 버튼에서 prefix별 태그 필터
- 서지정보 탭의 문서목록/서지정보 카드는 접기/펼치기

## 3. DESIGN 콘텐츠

DESIGN은 불교 지식을 시각화하는 인포그래픽, 도식, 삽화, 스타일시트 라이브러리입니다.

현재 DESIGN은 `src/content/design/{slug}/index.md`와 `assets/` 폴더를 함께 둡니다. `public/design`에 직접 원본을 넣지 않습니다.

### 3.1 폴더 구조

```
src/content/design/
└── two-perspectives/
    ├── index.md
    └── assets/
        ├── index.html
        ├── preview1.png
        └── thumb.jpg
```

빌드 후:

```
public/generated/design/two-perspectives/index.html
public/generated/design/two-perspectives/preview1.png
public/generated/design/two-perspectives/thumb.jpg
```

상세 페이지 URL:

```
/design/two-perspectives/
```

### 3.2 DESIGN index.md 예시

```yaml
---
title: "불교의 세계관 — 두 가지 관점"
description: "주관적 관찰과 객관적 관찰을 통해 불교적 세계관을 설명하는 인포그래픽."
summary: "두 가지 관점으로 정리한 불교 세계관"
primaryKind: infographic
type: diagram
format: mixed
series: "불교의 세계관"
version: "0.1.0"
status: published
date: 2026-06-11

htmlAsset: "index.html"
thumbnailAsset: "thumb.jpg"
imageAsset: "preview1.png"
previewAssets:
  - "preview1.png"
imageAlt: "두 가지 관점에서 나와 세계를 이해하는 불교의 세계관 인포그래픽"

pageSize: "A4"
orientation: portrait
medium: "HTML/CSS, Image"
source: "OL Project"
credits: ["비움"]
license: "CC0"

prefixTags:
  - "kind:infographic"
  - "topic:불교세계관"
  - "format:html"
  - "format:image"
tags:
  - 불교세계관
  - 두 가지 관점
  - 인포그래픽
published: true
---
```

### 3.3 DESIGN 필드 설명

| 필드 | 설명 |
|------|------|
| `title` | 자료 제목 |
| `description` | 목록·SEO 설명 |
| `summary` | 짧은 요약 |
| `primaryKind` | `infographic`, `illustration`, `style-sheet`, `reference` |
| `type` | `diagram`, `flowchart`, `comparison`, `poster` 등 |
| `format` | `html`, `pdf`, `image`, `mixed` |
| `series` | 연작 이름 |
| `htmlAsset` | `assets/` 안의 HTML 파일명 |
| `pdfAsset` | `assets/` 안의 PDF 파일명 |
| `thumbnailAsset` | 목록 카드 썸네일 |
| `imageAsset` | 대표 이미지 |
| `previewAssets` | 상세 미리보기 이미지 목록 |
| `imageAlt` | 대체 텍스트 |
| `credits` | 제작자. 직접 만든 자료는 `비움` |
| `license` | 라이선스 |
| `tags` | 표시용 태그 |
| `prefixTags` | 관리용 prefix 태그 |
| `published` | 공개 여부. 기본값은 `false` |

### 3.4 type 분류표

| 값 | 의미 |
|----|------|
| `infographic` | 인포그래픽 |

| `diagram` | 도식 |
| `map` | 지도 |
| `comparison` | 비교표 |
| `flowchart` | 흐름도 |
| `concept-map` | 개념 지도 |
| `poster` | 포스터 |
| `illustration` | 삽화 |
| `style-sheet` | 스타일시트 |
| `turnaround` | 턴어라운드 |
| `portrait` | 인물·도상 |
| `costume` | 복장·가사 |
| `architecture` | 건축·사찰 |
| `landscape` | 지역·자연 |

| `artifact` | 유물·공예 |

## 4. BLOG 콘텐츠

BLOG는 OL의 작업 일지, 개발 기록, 제작 노트입니다.

### 4.1 폴더 구조

```
src/content/blog/my-post/
└── index.md
```

폴더명이 URL slug가 됩니다.

```
src/content/blog/my-post/index.md
→ /blog/my-post/
```

### 4.2 BLOG index.md 예시

```yaml
---
title: "TEXT와 STORY 리더 개편"
description: "TEXT·STORY 콘텐츠 구조를 시리즈 중심으로 정리한 작업 기록."
date: 2026-06-11
category: OL
readingTime: 8
tags: ["개발", "TEXT", "STORY"]
prefixTags:
  - "kind:blog"
  - "project:ol-home"
  - "topic:development"
published: true
---
```

BLOG의 `published` 기본값은 `false`입니다. 공개하려면 반드시 `published: true`를 적습니다.

이미지는 포스트 폴더 안에 넣고 상대 경로로 참조합니다.

```markdown
![스크린샷](./screenshot.png)
```

시간순으로 흐르지 않는 브랜드 문서, 선언문, 안내 문서는 `src/content/pages/`에 둡니다.

title: "OL 브랜드"

푸터나 네비게이션에 연결하려면 관련 컴포넌트를 수정해야 합니다.

## 6. ATLAS

파일명은 반드시 다음 형식을 따릅니다.

ol-atlas_v{버전}.html

```
public/atlas/latest/ol-atlas_v0.0.7.html
```

빌드 시 버전과 파일 크기를 자동으로 읽어 ATLAS 페이지에 반영합니다.

## 8. 자주 하는 실수와 해결법

### 콘텐츠가 목록에 보이지 않음

`published: true`를 확인합니다.

기본값:

| 컬렉션 | 기본값 |
|--------|--------|
| blog | `false` |
| text | `false` |
| design | `false` |
| story | `true` |
| pages | `true` |

### DESIGN 자산이 보이지 않음

다음을 확인합니다.

- 자산이 `src/content/design/{slug}/assets/` 안에 있는가
- `index.md`의 `htmlAsset`, `thumbnailAsset`, `imageAsset`, `previewAssets`가 실제 파일명과 일치하는가
- `npm run build` 또는 `npm run sync:assets`를 실행했는가

### STORY HTML 자산 링크가 깨짐

다음을 확인합니다.

- 자산이 `src/content/story/{series}/assets/` 안에 있는가
- `htmlAsset` 값이 `assets/` 안 실제 파일명과 일치하는가
- 빌드 후 `public/generated/story/{series}/`에 파일이 복사되었는가

### TEXT/STORY 태그 필터가 이상함

태그 형식을 확인합니다.

```yaml
tags:
  - "인물/붓다"
  - "장소/룸비니"
  - "개념/연기"
```

`인물:붓다`처럼 콜론을 쓰면 TEXT/STORY의 prefix 필터 기준과 맞지 않습니다. TEXT/STORY의 사용자 필터용 태그는 `/`를 사용합니다.

### 빌드 오류: 스키마 불일치

frontmatter 필드명이 `src/content.config.ts`의 스키마와 정확히 일치하는지 확인합니다.

```yaml
# 잘못된 예
Title: "..."       # title이어야 함
publish: true      # published이어야 함
```

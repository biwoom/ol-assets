# OL 홈페이지 마크다운 문법 가이드

> **대상**: OL 콘텐츠 작성자
> **적용 범위**: `src/content/` 아래 모든 `.md` / `.mdx` 파일
> **기술 기반**: Astro 6 + remark-gfm

---

## 1. 기본 문법

### 1.1 헤딩

```markdown
## 대제목 (h2)
### 중제목 (h3)
#### 소제목 (h4)
```

- `# h1`은 페이지 제목으로 자동 생성되므로 **본문에서 사용하지 않습니다**.
- `## h2` 부터 사용합니다.

### 1.2 텍스트 강조

```markdown
**볼드 텍스트**
*이탤릭 텍스트*
***볼드 이탤릭***
~~취소선~~
```

### 1.3 링크

```markdown
[표시 텍스트](https://example.com)
[내부 링크](/works/buddha-story/01-birth)
```

### 1.4 이미지

```markdown
![대체 텍스트](./image-file.jpg)
```

이미지 파일은 **같은 폴더에 저장**하고 `./`로 시작하는 상대 경로를 사용합니다.

### 1.5 리스트

```markdown
- 순서 없는 리스트
- 항목 2
  - 중첩 항목

1. 순서 있는 리스트
2. 항목 2
```

### 1.6 인용

```markdown
> 인용문입니다.
> 여러 줄도 가능합니다.
```

### 1.7 코드

인라인 코드:

```markdown
`inline code`
```

코드 블록:

````markdown
```javascript
const x = 1;
```
````

### 1.8 테이블

```markdown
| 항목 | 설명 |
|------|------|
| A    | 설명 A |
| B    | 설명 B |
```

### 1.9 수평선

```markdown
---
```

## 2. 각주 (Footnote)

`remark-gfm` 플러그인으로 각주를 지원합니다.

### 2.1 기본 사용법

본문에 `[^번호]`를 넣고, 파일 하단에 `[^번호]: 설명`을 적습니다.

```markdown
붓다는 보리수 아래에서 깨달음을 얻었다[^1]. 
이후 사르나트에서 최초 설법을 했다[^2].

[^1]: 《니다나카타》 참조. 보리수는 현재 보드가야에 있다.
[^2]: 초전법륜(初轉法輪). 다섯 비구에게 사성제를 설했다.
```

### 2.2 결과

- 본문: 윗첨자 `1`, `2` 링크가 표시됩니다.
- 페이지 하단: "각주" 구분선 아래에 `[1]`, `[2]` 각주 목록이 나타납니다.
- 클릭: 본문 → 각주, 각주 `↩` → 본문으로 상호 이동합니다.
- 클릭 시 해당 각주 배경이 하이라이트됩니다.

### 2.3 주의사항

- 각주 번호는 숫자뿐 아니라 문자열도 가능합니다: `[^note-a]`
- 각주 정의(`[^1]: ...`)는 파일 어디에 있어도 되지만, **파일 맨 아래에 모아두는 것**을 권장합니다.
- 각주 정의와 본문 사이에 빈 줄이 있어야 합니다.

## 3. HTML 태그 직접 사용

`.md` 파일 안에서 HTML 태그를 **추가 설치 없이** 바로 사용할 수 있습니다.

### 3.1 자주 쓰는 HTML 태그

```html
<!-- 볼드 (CJK 문자 호환 문제 해결용) -->
<strong>도솔천(兜率天)</strong>

<!-- 하이라이트 -->
<mark>강조할 텍스트</mark>

<!-- 접기/펼치기 -->
<details>
<summary>더 보기</summary>

여기에 접힌 내용을 적습니다. 마크다운 문법도 사용 가능합니다.

- 리스트 항목
- **볼드**도 됩니다

</details>

<!-- 줄바꿈 -->
첫 번째 줄<br>두 번째 줄

<!-- 윗첨자 / 아랫첨자 -->
H<sub>2</sub>O
E = mc<sup>2</sup>

<!-- 작은 글씨 -->
<small>부가 설명</small>
```

### 3.2 주의사항

- HTML 블록 태그(`<div>`, `<details>`, `<table>` 등) 앞뒤에 **빈 줄**이 있어야 합니다.
- HTML 블록 안에서 마크다운 문법을 쓰려면 빈 줄로 구분해야 합니다.

```markdown
<details>
<summary>제목</summary>

여기부터 마크다운이 적용됩니다.

**볼드**, *이탤릭* 모두 작동합니다.

</details>
```

빈 줄 없이 바로 적으면 마크다운이 적용되지 않고 일반 텍스트로 출력됩니다.

## 4. 알려진 제약 사항

### 4.1 CJK + 괄호 + `**` 볼드 문제

마크다운 파서(remark)는 CJK 문자(한자, 일본어 등)가 닫는 괄호 `)` 바로 뒤에 `**`가 오면 볼드를 인식하지 못합니다.

```markdown
<!-- ❌ 실패 — 한자 + 닫는 괄호 + ** -->
**도솔천(兜率天)**

<!-- ✅ 성공 — 한글 + 닫는 괄호 + ** -->
**선택에 가까운 서원(誓願)의 실현**
```

차이: 실패 케이스는 `天)`에서 CJK 문자가 `)` 직전에 오고 바로 `**`가 닫힙니다. 성공 케이스는 `의 실현)`처럼 한글이 `)` 앞에 옵니다.

**해결 방법**: 이 패턴이 발생하면 `<strong>` HTML 태그를 직접 사용합니다.

```html
<strong>도솔천(兜率天)</strong>에서 시작한다.
```

### 4.2 해당 패턴 정리

| 패턴 | 결과 | 대처 |
|------|------|------|
| `**한글텍스트**` | ✅ 정상 | 그대로 사용 |
| `**영문(English)**` | ✅ 정상 | 그대로 사용 |
| `**텍스트(한자)의 설명**` | ✅ 정상 | 그대로 사용 |
| `**텍스트(한자)**` | ❌ 실패 | `<strong>` 사용 |
| `**텍스트(Pāli)**` | ❌ 실패 | `<strong>` 사용 |

규칙: **닫는 `)**` 직전이 비ASCII 문자이고 `**` 바로 뒤에 비단어 문자가 오면 실패**합니다.

## 5. .mdx 파일

확장자를 `.mdx`로 바꾸면 Astro 컴포넌트를 임포트해서 사용할 수 있습니다.

```mdx
---
title: "제목"
---
import MyComponent from '../../components/MyComponent.astro';

일반 마크다운 본문입니다.

<MyComponent prop="value" />

마크다운을 계속 적을 수 있습니다.
```

현재 OL에서는 `.md`만 사용하고 있으며, `.mdx`는 필요할 때 전환하면 됩니다.

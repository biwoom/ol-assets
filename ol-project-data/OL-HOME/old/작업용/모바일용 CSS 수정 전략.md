# 모바일용 CSS 수정 전략

---

## 헤더 — 모바일 전략

현재 문제: 900px 이하에서 `.ol-nav { display: none }`, 햄버거 메뉴 없음. 네비게이션이 완전히 사라집니다.

**권장**: 하단 시트(bottom sheet) 방식 모바일 메뉴

햄버거 아이콘 → 하단에서 올라오는 메뉴판. 이유:

- 상단 드로어(위에서 내려오는)는 WORKS 사이드바와 z-index 충돌 위험
- 하단 시트는 엄지 접근성이 좋음 (모바일 UX 표준)
- shadcn/Basecoat의 sheet 패턴과 일치

메뉴 구성: 6개 링크(HOME~AI) + BLOG + GitHub를 세로 나열. 검색 버튼과 다크모드 토글은 헤더에 남겨둠(아이콘 2개는 공간 부담 없음).

헤더 높이: 64px → 모바일에서도 64px 유지 권장. 줄이면 터치 타겟이 작아져서 불편합니다.

---

## 브레이크포인트 정리

현재 3단계(1100/900/600)인데 역할이 겹치는 부분이 있습니다.

**권장 정리**:

```
1100px — WORKS TOC 숨김 (현재 그대로)
768px  — 태블릿 전환 (900px → 768px로 조정 권장)
         헤더 네비 숨김 + 햄버거 표시
         그리드 2컬럼 → 1~2컬럼
480px  — 소형 폰 (600px → 480px로 조정 권장)
         모든 그리드 1컬럼
         히어로 타이포 최소화
```

900px와 600px는 현대 기기 기준으로 약간 어중간합니다. 768px(iPad mini/표준 태블릿)과 480px(소형 폰)이 실제 기기에 더 잘 맞습니다. 다만 기존 코드가 900/600으로 되어 있으므로, 변경 범위를 줄이려면 900/600을 유지하되 문제가 있는 곳만 768 추가하는 것도 방법입니다.

## 페이지별 권장안

### HOME

**히어로**:
- `h1` 64px → 600px에서 32px는 적절. 중간(768px)에 40px 단계가 있으면 좋음
- `.ol-lead` 18px → 모바일 16px로 줄이고 `max-width: 62ch` 제거 (화면 가득 채우기)
- CTA 버튼 2개: 모바일에서 `flex-direction: column`, 가로 전체 너비

**Product Cards**:
- `.ol-grid-3` → 모바일에서 1컬럼은 맞음
- `min-height: 240px` → 모바일에서 제거. 내용 길이에 따라 자연스럽게

**Stats**:
- 4개 항목, 현재 모바일 2×2 → 유지하되 숫자 `font-size`를 줄여야 함 (48px급이면 28px로)

**Manifest (다크 섹션)**:
- `padding: 48px` → 모바일 `padding: 32px 20px`
- 내부 텍스트 `max-width` 제거

**Latest Updates**:
- `.ol-grid-3` → 모바일 1컬럼, 카드 간 간격 줄이기

### ATLAS

- 인라인 `max-width: 48ch/50ch` → 모바일에서 무시되도록 `max-width: 100%` 오버라이드
- 피처 그리드가 있다면 1컬럼으로

### BOOK

- 카드 `.ol-grid-2` → 1컬럼
- "웹에서 읽기" + "내려받기" 버튼: `flex-direction: column`, 전체 너비

### WORKS

**가장 큰 문제**: `min-width: 1200px`과 `min-width: 900px`

- `min-width` 두 개 모두 **제거**. 이것이 모바일에서 가로 스크롤을 유발하는 직접 원인
- 모바일에서 사이드바는 이미 오버레이(fixed)로 전환되므로 `min-width`는 불필요
- `.ol-works-main padding: 48px 56px` → 모바일 `24px 16px`은 현재 맞음
- TOC: 1100px 이하 숨김 현재 유지

### BLOG

- `.ol-post grid: 220px 1fr` → 모바일 1컬럼은 맞음
- `h3 font-size` 미조정 지적이 보고서에 있음 → 모바일에서 20px 이하로 조정
- `blog/[slug].astro`의 인라인 `max-width: 720px` → 모바일에서 `max-width: 100%`

### DESIGN

- `design/[slug].astro`의 `max-width: 800px` → 모바일에서 100%
- 이미지 그리드 → 1컬럼

### AI

- 채팅 버블 `max-width: 80%/90%` → 모바일에서 `95%/100%`로 확대

### Entity

- `.ol-entity-list` 3컬럼 → 모바일 1컬럼 (현재 600px에서 적용됨, 적절)

## 공통 — 인라인 스타일 처리 전략

보고서에서 지적한 인라인 `max-width`들(6개 페이지)이 문제입니다. 인라인 스타일은 미디어쿼리로 오버라이드할 수 없습니다.

**권장 2가지 중 택1**:

**A. 클래스로 이관** (정석): 인라인 `max-width: 48ch` 등을 `.ol-prose-narrow` 같은 클래스로 교체 후 미디어쿼리에서 `max-width: 100%` 적용. 깔끔하지만 여러 파일 수정.

**B. `!important` 허용** (빠른 해결): 미디어쿼리에서 `[style*="max-width"] { max-width: 100% !important; }` 같은 패턴. 빠르지만 유지보수 부채.

**A를 권장**합니다. 지금 인라인 스타일이 6곳뿐이라 이관 비용이 크지 않고, 나중에 늘어나면 B가 걷잡을 수 없어집니다.

## 우선순위 정리

```
1순위: min-width 제거 (WORKS 가로 스크롤 즉시 해결)
2순위: 헤더 모바일 메뉴 (네비게이션 소실 해결)
3순위: 인라인 max-width → 클래스 이관
4순위: 히어로/Stats 타이포 단계 조정
5순위: 버튼/카드 모바일 레이아웃 세부 조정
```

1순위는 버그 수정이고 2순위는 기능 누락이므로 이 두 가지를 먼저 진행하는 것을 권장합니다. 3~5순위는 그 다음에 일괄 처리해도 됩니다.

# 응답

핵심 원칙부터 말하면, 이 프로젝트에서:

* AI는 “대량 처리 · 비교 · 정렬 · 초안 생성”
* 당신은 “정전성 판단 · 의미 선택 · 문체 결정 · 불교적 해석”

을 담당해야 합니다.

즉 이 프로젝트는 단순 AI 자동화가 아니라:

> “AI-assisted Buddhist Editorial System”

에 가깝습니다.

특히 불교 문헌 작업에서는:

* 무엇을 남길 것인가
* 무엇을 축약할 것인가
* 어떤 용어를 선택할 것인가
* 어떤 분위기로 전달할 것인가

가 단순 언어 문제가 아니라 “법(法)에 대한 이해”와 연결됩니다.

이 부분은 현 단계 AI가 안정적으로 수행하기 어렵습니다.

따라서 가장 좋은 구조는:

```text id="g4djlwm"
AI = 대량 처리 + 정렬 + 초안
인간(당신) = 방향성 + 의미 판단 + 최종 승인
```

입니다.

아래처럼 구분하는 것을 추천합니다.

# [0] Canonical Spine 구축

## AI 역할

AI는:

* 기존 목차 분석
* 사건 순서 추출
* chapter 후보 생성
* 중복 사건 탐지
* timeline 정렬

을 수행.

예:

```text id="58whfx"
출가 이전
출가
고행
깨달음
초전법륜
교단 형성
```

같은 구조 자동 정리.

---

## 당신 역할 (핵심)

여기는 반드시 당신 중심이어야 합니다.

왜냐하면 이 단계는:

* “어떤 붓다 서사를 만들 것인가”
* “어떤 세계관을 기준으로 삼을 것인가”

를 결정하기 때문입니다.

예:

* 대승적 신화 강조?
* 인간 붓다 중심?
* 수행 과정 강조?
* 일반 대중 친화?
* 역사성 우선?

이것은 단순 편집 문제가 아니라:

> 작품 철학 결정

입니다.

AI는 보조만 해야 합니다.

# [1] 원전 구조 분해 및 segment 단위화

## AI 역할 (매우 강함)

이건 AI가 매우 잘합니다.

예:

* 문단 분해
* 사건 단위 clustering
* parallel alignment
* source tagging
* segment naming

등.

---

## 당신 역할

검수 + 구조 수정.

특히:

* AI가 잘못 묶은 segment 수정
* 삽입 게송 처리
* 반복 구문 처리
* 문맥 경계 수정

이 중요합니다.

---

# [2] 사건 ID(Event ID) 부여

## AI 역할

AI가:

* 규칙 기반 ID 자동 생성
* 중복 검사
* relation map 생성

가능.

---

## 당신 역할 (중요)

“사건의 독립성 판단”

예:

```text id="c9x6yr"
마야부인 꿈
↕
도솔천 관찰
```

을:

* 하나 사건으로 볼지
* 분리할지

는 인간 판단 필요.

이건 narrative ontology 설계입니다.

---

# [3] 번역 메모리 + 용어집 구축

## AI 역할 (매우 중요)

AI 최적 분야 중 하나.

가능한 작업:

* 용어 추출
* 병렬 비교
* 등장 빈도 계산
* inconsistent translation 탐지
* source alignment

예:

```text id="m1jtl5"
bodhisatta
→ 보살
→ 보리살타
→ 미래불
```

자동 수집 가능.

---

## 당신 역할 (핵심)

여기가 당신의 가장 중요한 역할 중 하나입니다.

왜냐하면:

> 용어 선택 = 사상 선택

이기 때문입니다.

예:

| 원어     | 번역  | 의미 차이 |
| ------ | --- | ----- |
| Dhamma | 법   | 전통적   |
| Dhamma | 진리  | 현대적   |
| Dhamma | 가르침 | 대중적   |

같은 차이가 발생.

또:

| 용어         | 효과       |
| ---------- | -------- |
| 보살         | 불교 내부 언어 |
| 깨달음을 구하는 이 | 대중 친화    |
| 보리살타       | 원어 보존    |

모두 다른 독서 경험을 만듭니다.

이건 AI가 결정하면 안 됩니다.

---

# [4] 1차 직역층 생성

## AI 역할 (주도 가능)

이건 AI가 잘함.

특히:

* literal translation
* syntax preservation
* 병렬 번역
* 원문 대응 유지

는 로컬 LLM이 매우 효율적.

---

## 당신 역할

직역 오류 검수.

특히:

* 불교 개념 오역
* 문맥 왜곡
* 주체 혼동
* 시제 문제
* 산스크리트/팔리어 의미 손실

검토.

---

# [5] 의미 기반 2차 번역

## AI 역할

AI는:

* 자연어화
* 현대어 변환
* 가독성 향상

매우 잘함.

---

## 당신 역할 (매우 중요)

여기서:

* 지나친 현대화 방지
* 수행적 분위기 유지
* 불교적 정조 유지

가 필요.

예:

```text id="n84lh4"
"nirvana"
→ 열반
→ 완전한 자유
→ 괴로움이 꺼진 상태
```

무엇을 선택하느냐는 작품 방향성 문제.

또:

* 너무 유튜브식 문체
* 과도한 감성화
* 현대 심리학식 번역

을 억제해야 함.

---

# [6] 원전 비교·충돌 분석

## AI 역할 (매우 강함)

AI 핵심 분야.

가능 작업:

* 차이점 추출
* 누락 탐지
* variant mapping
* contradiction detection

예:

```text id="n8x7zi"
불본행집경:
칠보 위를 걸음

GCB:
즉시 선언
```

자동 비교 가능.

---

## 당신 역할 (핵심)

“무엇을 최종본에 반영할 것인가”

결정.

이건 단순 사실 문제가 아닙니다.

예:

* 신화성 유지?
* 역사성 강화?
* 대중성 우선?
* 수행성 강조?

철학적 선택이 개입됩니다.

---

# [7] Narrative Merge

## AI 역할

AI는:

* 중복 제거
* 흐름 연결
* 구조 재배열
* 초안 생성

잘함.

---

## 당신 역할 (매우 중요)

여기서부터는:

> “번역 프로젝트”
> 가 아니라
> “작가·편집자 작업”

입니다.

즉:

* 어디서 호흡 끊을지
* 어떤 장면 강조할지
* 어디를 침묵으로 둘지
* 무엇을 삭제할지

를 결정.

OL 선언문이 말한:

> “깊은 체류”
> “몰입”
> “수행적 독서”

가 여기서 결정됩니다. 

---

# [8] 현대 한국어 literary polishing

## AI 역할

AI는:

* 문체 균질화
* 반복 감소
* 리듬 정리

가능.

---

## 당신 역할 (절대 핵심)

이건 거의 인간 영역입니다.

왜냐하면:

* 문체
* 여백
* 침묵
* 수행적 호흡
* 정서적 밀도

는 아직 인간 편집자의 영역에 가깝기 때문.

특히 당신 프로젝트는:

* 단순 정보서가 아니라
* 수행적 narrative

를 지향합니다. 

따라서:

* 지나친 설명 제거
* 문장의 숨결 조정
* 독서 호흡 설계

는 당신이 해야 합니다.

---

검토.

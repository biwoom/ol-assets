## OL 브랜드 정의서

**Version 1.1 · 2026.06.12**

이 문서는 OL 프로젝트의 브랜드 정체성을 정의합니다. OL을 함께 만들어가는 모든 참여자가 같은 마음을 품기 위한 약속이며, 모든 기술적·디자인적·콘텐츠적 결정의 기준점입니다.

---

## 1. 브랜드 개요

### 1.1 브랜드명

| 항목 | 값 |
|------|------|
| 한글 | 올 |
| 영문 | OL |
| 발음 | [올] / [ol] |

### 1.2 핵심 정의

OL은 불교 경전과 논서를 디지털 환경에 최적화된 인터랙티브 웹북으로 재구축하는 불교 콘텐츠 스튜디오입니다. 백과사전식 나열을 지양하고, 문맥과 개념이 유기적으로 연결된 살아있는 학습 도구를 지향합니다.

### 1.3 태그라인과 슬로건

**태그라인**
- 한국어: *실처럼 이어진 지혜*
- 영문: *Thread by thread, truth unfolds*

**슬로건**
- *지혜의 올을 짜다* (Weaving the Wisdom)

## 2. 이름의 의미 구조

"올"이라는 한 음절에는 세 개의 층위가 겹쳐 있습니다.

### 층위 1 — 올(실의 올, 직물의 세로줄)

섬세함, 디테일, 텍스트의 기본 단위.

경전(經典)의 '經'은 날실을 뜻합니다. 산스크리트어 'sūtra'는 실, 줄을 의미하며 부처의 말씀을 실처럼 꿰어 놓았다는 뜻입니다. 인연의 실로 모든 존재가 연결되어 있다는 연기법(緣起法), 선업과 악업의 올이 모여 현생의 천을 이루는 업보 사상까지 — 불교의 핵심 세계관이 이 한 글자에 담겨 있습니다.

```
날실(經) = 경전(經典)
씨실(緯) = 논서(論書)
올 하나하나 = 개별 개념, 문장, 단어
천 전체 = 법계(法界)의 총체
직조 행위 = 학습과 깨달음의 과정
```

### 층위 2 — 오르다(상승, 향상)

위로 향함. 상구보리(上求菩提) — 위로 깨달음을 구한다. Deep Dive의 역설: 깊이 파고들수록 높이 올라갑니다. 무명(無明)에서 명(明)으로의 상승.

### 층위 3 — 올(모두, 전부)

일체(一切), 법계(法界). 부분이면서 동시에 전체를 포괄합니다.

### 인드라망(因陀羅網)과의 일치

화엄경의 인드라의 그물과 정확히 일치합니다. 각 올이 다른 모든 올과 연결되고, 하나를 당기면 전체가 움직입니다(연기법). 부분과 전체가 상즉상입(相卽相入)합니다. OL의 온톨로지 기반 지식그래프가 지향하는 구조 그 자체입니다.

## 3. 존재 이유

OL의 출발점은 기술이 아니었습니다.

> *"나는 왜 불교 콘텐츠를 만들고 싶은가?"*

그 답은 법(法)을 만났을 때의 기쁨, 괴로움의 감소, 자비와 이타심, 존재에 대한 통찰, 명상을 통한 고요함과 자유 — 이런 것들을 다른 사람들과 나누고 싶다는 마음이었습니다.

따라서 OL의 핵심은 *정보 전달*이 아니라 **고통 감소와 삶의 전환을 돕는 경험**입니다.

모든 결정의 기준이 되는 단 하나의 질문:

> ***"사람들이 이 경험을 통해 정말 더 자유로워지는가?"***

## 4. 브랜드 철학

### 4.1 핵심 가치

**연결(Connection)** — 올 하나하나가 모여 천을 이루듯, 고립된 텍스트들을 살아있는 연결망으로 엮습니다. 모든 개념, 경전, 논서는 독립적이면서도 서로 연결되어 있습니다.

**깊이(Depth)** — 천의 표면을 보는 것이 아니라, 올 하나하나를 따라가며 깊이 탐구합니다. 백과사전식 나열이 아닌, 단일 주제에 대한 심층적 이해를 추구합니다.

**여정(Journey)** — 각자가 자신의 올을 선택해 따라가는 학습 여정입니다. 같은 천이지만 각자 다른 경로로, 사용자가 직접 자신만의 천을 짜는 경험을 제공합니다.

### 4.2 디자인 철학

**최소주의** — 한 음절, 두 글자의 절제된 아름다움. 덜어낼수록 더 많이 담는다는 동양 미학.

**유기성** — 정적 데이터베이스가 아닌 살아 숨쉬는 연결망. 사용자의 학습에 따라 성장하고 변화하는 구조.

**개방성** — 로그인 장벽 없이 즉시 접근 가능. 지식은 모두에게 열려 있어야 한다는 신념. 사용자 데이터의 완전한 통제권 보장.

### 4.3 문사수(聞思修)

OL은 지식 관리 시스템이 아니라 **문사수 환경**에 가깝습니다.

- **문(聞)** — 불교 이야기를 읽고 듣는다. 최소한의 연결이 맥락과 의미의 흐름을 돕는다.
- **사(思)** — 질문하고, 사유하고, 연결하고, 삶과 대조한다.
- **수(修)** — 명상하고, 실천하고, 삶 속에서 체화한다.

## 5. 브랜드 체계

OL은 하나의 앱이 아니라 역할이 분리된 체계입니다.

### 5.1 제품 구조

| 제품            | 역할     | 설명                                            |
| ------------- | ------ | --------------------------------------------- |
| **OL HOME**   | 포털     | 정적 웹사이트. 모든 제품의 입구이자 소개                       |
| **OL ATLAS**  | 도구     | 단일 HTML 파일로 동작하는 콘텐츠 뷰어 겸 에디터. 현재 개발 일시 보류상태. |
| **OL STORY**  | 서사콘텐츠  | 서사 중심 이야기불교 콘텐츠. OL BOOK 대치.                  |
| **OL TEXT**   | 문헌콘텐츠  | 번역, 주석, 연구 노트를 중심 문헌콘텐츠. OL WORKS 대치.         |
| **OL DESIGN** | 시각 콘텐츠 | 불교 지식을 시각화하는 인포그래픽과 보조 삽화·스타일시트               |
| **OL BLOG**   | 작업일지   | 콘텐츠개발과정 및 코드수정과정 기록                           |

### 5.2 저장소 구조

도구와 콘텐츠를 분리하는 것이 핵심입니다.

```
OL-PROJECT (GitHub Organization)
├── biwoom.github.io → 홈페이지 포털 (Astro + GitHub Pages)
├── ol-home         → 개발용 홈페이지
├── ol-atlas        → 앱 엔진 소스코드
└── ol-assets       → 공용 시각 자산 라이브러리
```

### 5.3 기술 원칙

- **단일 파일 배포**: ATLAS와 BOOK은 단일 HTML로 배포됩니다. 설치도 서버도 필요 없습니다.
- **네트워크 독립**: 오프라인에서 동작합니다. 인터넷 연결이 필요 없습니다.
- **로컬 우선**: 계정 없음. 추적 없음. 모든 데이터는 사용자의 브라우저와 파일 안에만 머뭅니다.
- **CC0 라이선스**: 모든 콘텐츠는 출처 표기 없이 자유롭게 사용할 수 있습니다.

## 6. 디자인 시스템

### 6.1 핵심 원칙

OL의 디자인 시스템은 shadcn/ui의 본질을 따릅니다: **사용자가 디자인 자체에 집중하지 않고 콘텐츠에 몰입**할 수 있게 만드는 것.

```
모던   → 무채색 베이스 + 정확한 1px 보더
심플   → 장식 제거, 충분한 여백, 명확한 위계
세련   → 미세한 그림자, 정밀한 간격, 일관된 radius
질리지 않음 → 개인 취향이 강한 컬러 사용 안 함
```

이 원칙은 존재 선언문의 지향과 맞닿습니다. 빠른 소비, 과잉 인터랙션, 자극적 UX를 하지 않습니다. 깊은 체류와 조용한 탐색을 위해서는 디자인이 투명해야 합니다.

### 6.2 색상

shadcn의 neutral 팔레트를 채택합니다. HSL 기반 토큰으로 라이트/다크 모드를 정의합니다.

| 용도 | 라이트 | 다크 |
|------|--------|------|
| 배경 | `#FFFFFF` | `#0A0A0A` |
| 본문 | `#171717` | `#E5E5E5` |
| 보조 텍스트 | `#525252` | `#A3A3A3` |
| 테두리 | `#E5E5E5` | `#2A2A2A` |

### 6.3 타이포그래피

장식적 폰트를 모두 제거하고 시스템 폰트 + Pretendard로 통일합니다. 한국어 본문의 가독성이 최우선입니다.

- **한국어 본문**: Pretendard, 15px, line-height 1.75
- **영문 라벨/코드**: Geist Mono, 대문자, letter-spacing 0.08em
- **제목**: letter-spacing -0.025em, font-weight 700

### 6.4 간격과 Radius

Tailwind 4-base 시스템을 따릅니다.

```
간격: 4px — 8px — 12px — 16px — 24px — 32px — 48px
Radius: 4px(sm) — 6px(md) — 8px(기본) — 10px(lg)
```

### 6.5 아이콘

인라인 SVG 아이콘을 직접 정의하여 사용합니다. 외부 아이콘 라이브러리에 의존하지 않습니다. 단일 HTML 배포 원칙과 일치합니다.

## 7. 브랜드 보이스

### 7.1 톤

- **명상적** — 서두르지 않고 깊이 생각하게 합니다
- **초대적** — 강요하지 않고 탐험을 권유합니다
- **지혜로운** — 가르치려 하지 않고 통찰을 나눕니다

### 7.2 언어 스타일

- **간결함**: 짧은 문장, 명확한 의미
- **시적임**: 은유와 비유의 자연스러운 사용
- **존중함**: 사용자를 학습자이자 탐험가로 대우

### 7.3 피해야 할 톤

교조적, 권위적, 과도하게 기술적, 상업적, 마케팅적 톤은 사용하지 않습니다.

## 8. OL이 하지 않는 것들

이 목록은 기술적 한계가 아니라 의지적 결정입니다. 가능하지만 하지 않습니다.

- 사용자 간 공유/협업 기능
- 게이미피케이션 (배지, 점수, 진도율, 연속 사용일)
- 푸시 알림, 학습 리마인더
- 소셜 로그인, 계정 시스템
- 사용 통계, 대시보드
- 추천 알고리즘
- AI의 능동적 개입
- 데이터 수집, 텔레메트리
- 광고, 후원 콘텐츠
- 평가 기능 (별점, 좋아요, 댓글)

모든 앱이 사용자를 더 오래 머물게 하려 할 때, OL은 사용자가 **자기 삶으로 잘 돌아가는 것**을 성취로 삼습니다. 이 어긋남이 OL의 정체성입니다.

---

## 9. 대상 독자

OL은 전문가를 위한 학술 플랫폼이 아닙니다. 가장 우선적으로 바라보는 대상은 평범한 일반 대중입니다.

- 불교를 처음 접하는 사람
- 삶의 의미를 찾고 있는 사람
- 마음의 고통을 줄이고 싶은 사람
- 수행에 관심은 있으나 어렵게 느끼는 사람
- 깊은 이야기를 원하지만 지루한 설명은 원치 않는 사람

따라서 OL의 콘텐츠는 쉽고, 명료하며, 아름답고, 집중 가능하며, 반복해서 다시 찾아오고 싶어야 합니다.

## 부록 — OL 결정 카드

OL 개발 중 어떤 결정 앞에 섰을 때 꺼내 보는 카드입니다.

> **이 결정이 다음을 만족하는가?**
>
> 1. 깊은 체류를 돕는가, 빠른 소비를 부추기는가?
> 2. 사용자의 사유를 돕는가, 방해하는가?
> 3. AI가 청하지 않은 개입을 하게 만드는가?
> 4. 사용자의 기록을 사용자에게 속하게 두는가?
> 5. 가르침으로부터의 자유로 이끄는가, 가르침에의 집착을 강화하는가?
> 6. 단순함을 늘리는가, 복잡함을 늘리는가?
> 7. 사용자가 OL에 더 오래 머물게 만드는가, 자기 삶으로 더 잘 돌아가게 만드는가?
>
> 하나라도 답이 어긋난다면, 다시 생각합니다.

# 태그활용 메타데이터 구축 원리

태그는 “많이 나누는 것”이 아니라 **최소 개수로 핵심 축만 잡고, 나머지는 자연어로 흡수**하는 구조가 가장 효율적.

---

# 1. 설계 원칙 (핵심 4가지)

1. **6개 이내 축만 사용**
2. **prefix(접두어)로 의미를 부여**
3. **자연어를 최대한 허용** (엄격한 코드 금지)
4. **후처리로 구조 생성 가능해야 함**

---

# 2. 최적 태그 체계 (권장 6축)

아래 6개만 사용하면 대부분의 불교 텍스트를 커버할 수 있습니다.

---

## ① #경전 (Source)

형식:

```text
#경전:DN1
#경전:SN12.2
#경전:법구경
```

설명:

* 니까야 / 아함 / 기타 경전 식별
* 너무 엄밀하게 안 써도 됨

---

## ② #인물 (Person)

```text
#인물:붓다
#인물:아난
#인물:사리불
```

설명:

* 서사, 설법 맥락 추적 핵심
* 후에 인물 네트워크로 확장 가능

---

## ③ #장소 (Place)

```text
#장소:사위성
#장소:기원정사
#장소:라자그리하
```

설명:

* 지도 기능과 직결
* 초기에는 단순 문자열이면 충분

---

## ④ #주제 (Theme)

```text
#주제:무상
#주제:연기
#주제:고
#주제:팔정도
```

설명:

* 교리 정리의 핵심 축
* 가장 많이 쓰게 될 태그

---

## ⑤ #유형 (Type)

```text
#유형:설법
#유형:대화
#유형:인연담
#유형:게송
#유형:비유
```

설명:

* 법구경, 아함경, 니까야 모두 커버
* 문서뷰 구성 시 매우 중요

---

## ⑥ #시기 (Time)

```text
#시기:성도후1년
#시기:열반직전
#시기:초기교단
```

설명:

* 타임라인 기능의 핵심
* 정밀 연도 불필요

---

# 3. 실제 입력 예시 (현실적인 수준)

---

## 예시 1: 디가니까야 브라흐마잘라경

```text
제목: 브라흐마잘라경 - 62견 해체

본문:
외도들의 다양한 견해를 분석하고 그 집착 구조를 밝힌다...

태그:
#경전:DN1
#인물:붓다
#장소:라자그리하
#주제:견해
#유형:설법
```

---

## 예시 2: 법구경 인연담

```text
제목: 탐욕으로 인해 싸운 두 무리

본문:
두 무리가 물을 두고 다투었을 때 붓다는...

태그:
#경전:법구경
#인물:붓다
#주제:탐욕
#유형:인연담
```

---

## 예시 3: 연기 교리 정리

```text
제목: 12연기의 구조

본문:
무명 → 행 → 식 → 명색...

태그:
#주제:연기
#유형:교리정리
```

👉 경전 정보 없이도 입력 가능 (속도 유지)

---

# 4. 중요한 설계 포인트

## (1) 모든 태그를 채우지 않는다

❌ 잘못된 방식:

```text
6개 다 채우기
```

✔ 올바른 방식:

```text
2~3개만 사용
```

---

## (2) 자연어 허용

```text
#주제:마음챙김
#주제:분별
```

👉 표준화는 나중 문제

---

## (3) 중복 허용

```text
#주제:무상
#주제:고
```

---

# 5. 향후 확장 방식 (핵심)

이 구조의 강점은 **자동 변환 가능성**입니다.

---

## (1) 타임라인 자동 생성

```js
if(tag.startsWith('시기:')) {
  timeline.push(...)
}
```

---

## (2) 지도 자동 생성

```js
if(tag.startsWith('장소:')) {
  map.addMarker(...)
}
```

---

## (3) 인물 네트워크

```js
if(tag.startsWith('인물:')) {
  graph.link(...)
}
```

---

# 6. 주류 vs 비주류 비교

## 주류 (Notion식)

* 필드 많음
* 입력 느림
* 구조 명확

---

## 당신 방식 (권장)

> **태그 기반 느슨한 구조 + 후처리 구조화**

---

# 7. 최소 입력 규칙 (강력 권장)

매 카드:

* 태그 최대 3개
* 10초 내 작성

---

# 8. 핵심 요약

> 태그는 “분류 도구”가 아니라
> **“미래 구조를 위한 힌트”**로 사용해야 한다

---

# 9. 한 줄 결론

> **6개 축(prefix 태그)만 유지하고, 나머지는 버려라**

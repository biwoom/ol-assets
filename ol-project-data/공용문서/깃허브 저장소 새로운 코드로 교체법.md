기존의 깃허브 저장소(Repository) 내용을 완전히 새로운 코드로 교체하여 업데이트하고 싶을 때 진행하는 과정입니다. 기존의 커밋 히스토리(이력)를 유지하면서 코드만 깔끔하게 바꾸는 표준적인 절차를 안내해 드립니다.

터미널을 열고 아래 순서대로 명령어를 입력하시면 됩니다.

### Step 1. 로컬에 저장소 다운로드 (Clone)

우선 수정할 저장소를 로컬 컴퓨터로 가져옵니다.

Bash

```
git clone <저장소_URL>
cd <다운로드된_폴더명>
```

- `cd` 명령어를 통해 다운로드된 프로젝트 폴더 내부로 반드시 이동해야 합니다.
    

### Step 2. 기존 코드 삭제 (주의 필요)

폴더 안의 기존 파일들을 지울 때, **`.git` 폴더는 절대 지우면 안 됩니다.** `.git` 폴더가 삭제되면 깃허브 저장소와의 연결 정보와 이전 이력이 모두 날아갑니다.

- **안전한 방법:** 윈도우 탐색기나 맥 Finder를 열어 해당 폴더로 들어간 뒤, 숨김 파일인 `.git` 폴더를 제외한 모든 파일과 폴더를 선택해 삭제(휴지통으로 이동)합니다.
    
- 터미널 명령어로 삭제하고 싶다면 아래 명령어를 사용합니다.
    
    Bash
    
    ```
    # 현재 폴더 안의 모든 파일/폴더 삭제 (단, .git 같은 숨김 폴더는 제외됨)
    rm -rf *
    ```
    

### Step 3. 새 코드 복사 및 붙여넣기

새로 올릴 코드 파일들을 복사하여 방금 비워둔 로컬 저장소 폴더 안에 그대로 붙여넣습니다.

### Step 4. 변경 사항 확인 및 파일 추가 (Stage)

터미널에서 코드 변경 상태를 확인하고, 새로 추가된 파일들을 깃의 추적 대상에 올립니다.

Bash

```
# 1. 현재 어떤 파일이 지워지고 새로 추가되었는지 상태 확인
git status

# 2. 변경된 모든 파일(삭제, 수정, 추가)을 스테이징 영역에 추가
git add .
```

### Step 5. 커밋 및 업로드 (Commit & Push)

변경 사항에 대한 설명을 남기고 깃허브 원격 저장소로 전송합니다.

Bash

```
# 3. 버전 생성 및 설명 작성
git commit -m "Replace existing code with new project code"

# 4. 깃허브 원격 저장소로 업로드
git push origin main
```

- **참고:** 저장소에 따라 기본 브랜치 이름이 `main`이 아니라 `master`일 수 있습니다. 만약 에러가 난다면 `git status`나 터미널 창에 표시된 브랜치 이름을 확인한 후 `git push origin master`로 입력해 보세요.

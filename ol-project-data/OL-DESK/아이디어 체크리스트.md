# 아이디어 체크리스트

AI에이전트 번역도량(翻譯道場) 개발 아이디어.

새로운 요청사항 및 아이디어:

1. 페이퍼클립 서버와 별도의 인간검수자 용 로컬 개발서버 제작(가칭 OL DESK SERVER)
2. 페이퍼클립 에이전트가 생산한 json 산출물을 토대로 Astro 기반 웹빌더 활용하여 대시보드 제작하고 가상서버로 실시간 번역상황 확인. 인간 검수자가 3차 번역물과 용어집 검토하여 확정하면 실시간으로 검수파일 업데이트 되고 페이퍼클립 에이전트가 이를 감지하여 3차 반영 작업 시작.
3. 이 Astro 기반 대시보드(OL DESK)는 인간 검수자를 위한 UI, UX 디자인, OL HOME 디자인 철학과 스타일 코드 계승.
4. OL DESK 핵심기능: (1).문단 ID 별로 원문, 1차번역, 2차번역, 3차번역을 각각 볼 수 있고 함께 볼수 있고 깃헙 코드 히스토리처럼 수정부분을 겹쳐서 볼 수 있음. (2). 신규용어, 신규각주를 볼 수 있고 확정과 보류, 페기 체크할 수 있음. (3). 초기 페이퍼클립 CEO에게 전달할 첫 번째 '초기 기획 task 문서' 와 각 에이전트에게 전달할 AGENTS.md,  HEARTBEAT.md, SOUL.md, TOOLS.md 를 프로젝트를 터미널에서 설치할 때 함께 생산하거나 데시보드에서 버튼 클릭으로 문서생산.
5. 기존 AI협업 워크플로우 매뉴얼과 튜토리얼 기반 아이디어를 이런 OL DESK 개발과 운영관점에서 다시 전면 수정한다. 그리고 하나의 새로운 OL DESK 개발기획안으로 정리한다.
6. 태그 에이전트 신설한다. perfix 태그로 6개 정도의 간단한 태드입력. 예시) 인물:붓다, 장소: 제따와나, 지역: 라자가하 등. 에이전트를 위한 태그규칙 정립한다.
7. 기본 매뉴얼에서 다른 에이전트에 통합할 만한 에이전트 검토한다. A2-참고번역분리-분명, B1-인간검수, B4-최종인계-회향 등 통합 검토. 페이퍼클립에서 이슈별로 작업단계와 채팅내용 기록이 남음. 중복되는 기능 검토.
8. 기존 매뉴얼에서 에이전트가 산출하는 문서들은 페이퍼클립에서 작동하는 issues 기능과 이슈 하위에서 채팅드으로 기록되는 작업등 페이퍼클립에서 담당할 수 있는 부분을 간과하고 비대해진 측면이 있음. 이를 페이퍼클립과의 연계 관점에서 다시 검토한다. 그리고 Astro 기반 대시보드(OL DESK)를 위한 산출물 관점에서 필요한 정보들을 검토한다.
9. 산출물은 aI에이전트 상호간 소통을 기계적 자료는 json으로 통일한다. 인간 검수자를 위한자료는 마크다운 파일로 생산한다. 그런데 Astro 기반 대시보드(OL DESK) 구상에서는 인간 검수자는 직접 문서를 보는 것이 아니라 개발서버로 표시된 브라우저 화면을 보기 때문에 이를 고려하여 최종 번역물 정도만 마크다운으로 생산하는 것도 검토한다.
10. 에이전트 명칭은 OL-Desk Agent Name.md 문서 방식으로 수정한다.

## AI협어 워크플로우 종류와 상태정보 변경

1. 1-2차번역 워크플로우 : A-관리매니저 주도 
2. 2차 휴먼 워크플로우 : OL-DESK 주도
3. 3차번역 워크플로우 : B-확정매니저 주도 
4. 3차 번역본 및 4차원고 수정 워크플로우 : 4차 원고로 완료된 번역본이라 할지라도 OL-DESK에서 본문, 용어, 각주, 태그 수정시 다시 3차가번역본이 되어 3차번역 워크플로우 진행. 원고 상태정보는 '3차번역 워크플로우 진행중'으로 변경됨.

## OL-DESK 레이아웃

대시보드 스타일. 참조사이트: https://ui.shadcn.com/create 
메인 메뉴와 페이지 : 각 페이지가 상단 헤더 메뉴로 구성. 중앙정렬.
- CREW : 개별 agent 활동평가. 사이드바에 개별 에이전트 목록. 본문에 전체 에이전트 활동기록. 사이드바 개별 에이전트 클릭이벤트 > 해당 에이전트 활동 로그와 감수자 평가 기록 창.
- 진행 : 원본문헌 리스트 페이지. 원본 문헌들의 워크플로우 진행상태 리스트. 사이드바 준비중, 진행중, 감수필요, 완료됨 상태 필터 버튼. 개별 문헌 클릭이벤트 > 해당 문헌 번역페이지로 이동.
- 번역 : 번역본 확인 및 수정 페이지. 1, 2, 3, 4차 번역본의 본문, 용어, 각주, 태그 수정. 실제 수정은 3, 4본만 가능. 번역본 별 탭 페이지. 
- 용어 : 개별 프로젝트 전체 누적 용어 및 신규용어 페이지, 개별 용어 추가, 수정, 삭제 기능.
- 각주 : 개별 프로젝트 전체 누적 각주 및 신규각주 페이지, 개별 각주 추가, 수정, 삭제 기능.
- 태그 : 개별 프로젝트 전체 누적 태그 및 신규각주 페이지, 개별 태그 추가, 수정, 삭제 기능.
- 관리 : 개별 agent 활동평가 정보와 OL-DESK 코드 업데이트 아이디어 담은 관리문서 생성, 이전 관리문서 저장소. 깃헙 업로드 기능. 이슈별로 리스트로 그룹화. 이슈별로 평가와 완료 체크 기능.
